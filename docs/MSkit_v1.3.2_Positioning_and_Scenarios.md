# MSkit v1.3.2 三方叙事定位与应用场景

> **定位**：MSkit 是面向民用/商用实景活动、演练、赛事、搜救和安保训练的 **黑 / 红 / 蓝三方智能规控平台**。  
> **核心叙事**：黑方统揽全局，红方负责搜索检测，蓝方负责防护覆盖；系统通过 PNT、GOD/COD、CCD、路径规划、态势地图和报告导出形成闭环。  
> **合规边界**：本项目只面向民用活动组织、模拟竞赛、搜救训练、安保巡检和户外团队协作；不得解释为军事指挥、武器化、打击排序或作战控制。

---

## 1. 项目重新定位

MSkit 不再仅被定义为“便携式边缘智能信息规控系统”，而应被定义为：

> **面向黑/红/蓝多方协作的实景智能规控平台。**

其产品价值不是单点 AI 检测，而是把实景活动中的三类角色组织到一个可视化、可裁判、可规划、可复盘的系统中：

| 角色 | 产品定位 | 核心价值 |
|------|----------|----------|
| 黑方 | 组织者 / 裁判 / 调度方 | 统揽全局、配置规则、查看 PNT、规划路径、裁决事件、归档报告 |
| 红方 | 搜索 / 检测 / 攻方 / 搜救小队 | 通过 GOD/COD 发现目标、变化和异常，并提交证据 |
| 蓝方 | 防护 / 守方 / 安保小队 | 通过 CCD 设计防护覆盖、巡查路线和响应方案 |

---

## 2. 项目目标

### 2.1 产品目标

1. 将无人机视频、图片、日志、队伍轨迹和手工标注统一到离线地图和 PNT 时间线中。
2. 支持黑方创建活动、配置区域、组织红蓝队伍、查看全局态势、裁判/调度和报告归档。
3. 支持红方执行搜索、检测、识别、变化发现和证据提交。
4. 支持蓝方执行 CCD 覆盖设计、保护区/禁入区设置、盲区分析和巡查路径建议。
5. 通过 MVP 先用笔记本 + 无人机/素材导入验证三方核心闭环，再升级到 Jetson、RTK、热成像、LoRa 和加固箱。

### 2.2 MVP 目标

MVP 不追求完整硬件工程实现，而集中展示三方核心能力：

| 方别 | MVP 核心功能 | 展示价值 |
|------|--------------|----------|
| 黑方 | PNT、全局地图、活动区域、红蓝状态、路径规划、报告 | 证明系统能组织、裁判、调度和复盘 |
| 红方 | GOD/COD、搜索轨迹、发现点、变化点、候选目标复核 | 证明系统能从无人机数据中产生可解释发现 |
| 蓝方 | CCD、保护区、禁入区、覆盖缺口、巡查路径 | 证明系统能生成防护/守方方案 |

## 6. MVP 场景优先级

MVP 路演建议选择两个演示脚本：

### Demo A：Hide and Seek 实景对抗赛

这是最适合融资路演的主脚本，三方关系最清楚：

1. 黑方创建活动区、边界、红蓝队伍和规则。
2. 红方导入无人机素材，运行 GOD/COD，生成发现点和变化点。
3. 蓝方设计隐藏/防护区域，运行 CCD，生成覆盖路线和盲区分析。
4. 黑方查看 PNT 全局态势，判断红方发现和蓝方防护质量，导出报告。

### Demo B：野外搜救

这是最适合社会价值和非游戏场景的备选脚本：

1. 黑方创建搜救区域、起点、禁入区和优先搜索区。
2. 红方导入无人机视频/图片/日志，运行 GOD/COD，生成疑似人员/动物/风险点。
3. 黑方基于 PNT 和路径规划重新分配搜索路线。
4. 导出搜救报告和证据包。

## 7. 对外叙事建议

推荐表述：

- “黑方是组织者、裁判和调度方。”
- “红方是搜索、检测和证据采集小队。”
- “蓝方是防护、安保和值守小队。”
- “MSkit 解决的是实景活动中的多方协作、态势可视化、路径规划、检测发现、覆盖设计和复盘报告。”

避免表述：

- 作战、敌我识别、打击目标、武器识别、火力、歼灭、压制等军事化措辞。
- 自动飞控、自动追踪、自动航线下发等可能引发无人机合规风险的措辞。
