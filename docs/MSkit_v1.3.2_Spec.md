# MSkit 项目规格文档 v1.3.2

> **项目代号**：MSkit  
> **全称**：Multi-Scenario Edge Intelligence Kit  
> **版本**：v1.3.2（2026年7月，文档同步修订）  
> **定位**：便携式边缘智能信息规控系统，服务于纯民用与商用场景  
> **文档状态**：内部评审稿 / Engineering Review Draft（不作为正式对外规格）

> **冻结条件**：完成供应商 RFQ、关键器件样品验证、法律合规审查、系统级功耗测试、认证预测试后，方可进入 v1.4 规格冻结。

## 2. 合规定位与目标场景

MSkit 严格面向纯民用/商用场景，所有功能设计应避免军用采购、武器化与作战用途表述。Milsim 场景可保留，但对外宣传应使用“民用模拟”“安全作业”“活动流程模拟”“A/B 队伍现场状态”等表述。

| 场景 | 核心需求 | 合规边界 | 优先级 |
|------|----------|----------|--------|
| Milsim / Airsoft / Paintball 活动 | A/B 队伍现场状态模拟、活动流程模拟、实时通信 | 避免军用采购、武器化、作战用途表述 | P1 |
| 野外生态观察与合法追踪 | 动物热源探测、轨迹追踪、区域监控 | 涉及狩猎用途时按目标地区法律启用功能限制；在禁止热成像辅助狩猎的地区，禁用相关检测/记录/追踪 | P2 |
| 野外作业安保与巡检 | 周界监控、人员定位、异常告警 | 隐私保护；监控区域设置明显标识 | P2 |
| 救援演练与灾害响应 | 人员搜救、热源定位、路径规划、资源调度 | 与应急部门协调；数据脱敏 | P1 |
| 户外探险与团队协调 | 团队定位、地形导航、安全预警 | 用户知情同意；位置数据本地优先 | P3 |

## 3. 硬件同步要求

### 3.1 计算与存储

| 变体 | 计算模块 | 启动介质 | 说明 |
|------|----------|----------|------|
| Backpack | Orin Nano 8GB | 256GB–1TB NVMe SSD | 模块无板载 64GB eMMC；microSD 仅作维护/恢复 |
| Team Base / Team Thermal（原 Command） | Orin NX 16GB | NVMe SSD，或载板侧 eMMC/SSD | 不写板载 64GB eMMC |
| Field Pro / Field Max（原 Command Pro/Max） | AGX Orin 32/64GB | 模块板载 64GB eMMC + 外部 NVMe | eMMC 仅对 AGX Orin 成立 |

软件栈锁定为 JetPack 6.2.1 + CUDA 12.6.10 + TensorRT 10.3 + cuDNN 9.3。

### 3.2 电池与运输

| SKU 层级 | 推荐化学体系 | 单块能量 | 运输口径 |
|----------|--------------|----------|----------|
| 标准民用版 | Li-ion 智能电池包或 LiFePO4 | 目标 <300Wh 优先 | 降低危险品运输复杂度 |
| 专业/工程样机 | 6S LiPo | Backpack 355Wh / Team 488Wh | >300Wh 默认按危险品运输管理 |
| 外接扩展电池 | LiFePO4 优先 | 视客户场景 | 按目标市场运输/仓储/保险流程确认 |

UN38.3 仅说明通过运输测试，不等于可随意空运、随身携带或普通快递运输。销售前必须确认目标市场的空运、陆运、仓储、保险和售后退换货流程。

### 3.3 防护与 EMC

IP65/IP67 为整机装配后的目标等级，不直接引用箱体厂商原始等级。所有屏幕、风扇、接口、键盘、散热口和线缆穿舱件开孔完成后，必须对整机做 IEC 60529 测试方可宣称。

MIL-STD-810H 仅作为设计验证方法；未经第三方实验室测试前，不得对外宣称 “MIL-STD-810H certified”。EMC 按 SKU 拆分：个人/户外消费 SKU 目标 Class B；工业/商用 SKU 可按 Class A。

### 3.4 热成像

删除 “Boson+ 640 Clone” 表述。国产替代热成像模组只能作为候选方案，需提供型号、出口分类、RoHS/REACH、长期供货承诺与同条件画质测试报告。不得承诺“性能为原版 80–90%”。

## 4. 通信、LoRa 与无人机

### 4.1 LoRa Mesh

LoRa Mesh 仅承载低速遥测：位置、状态、告警、短文本。文件传输不得走 LoRa，改由 Wi-Fi Direct、以太网、USB、蜂窝网络或本地热点完成。

| 数据类型 | 建议频率 | 最大载荷 |
|----------|----------|----------|
| 位置广播 | 0.2–1Hz 可配置 | ≤64B |
| 节点状态 | 0.1–0.2Hz | ≤64B |
| 告警 | 事件触发 | ≤128B |
| 短消息 | 人工触发 | ≤240B |
| 文件 | 禁止走 LoRa | 改走 Wi-Fi/USB/以太网 |

LoRa、Wi-Fi 6E、LTE、蓝牙、GNSS、卫星通信均按目标市场锁定地区化 SKU。不得写“868/915MHz 通用”。

### 4.2 无人机集成

无人机集成为 Phase 2 功能，非 MVP 验收项。MVP 仅支持 DJI 飞行日志、媒体文件导入，以及实时截图/视频流桥接至 MSkit 地面显示。

| 接口 | 协议 | 用途 |
|------|------|------|
| 无人机数据接入（Phase 2） | MAVLink 2.0 / DJI 日志 / 视频流桥接 | 遥测读取、媒体导入、视频显示；飞控级控制需 PoC 后单独冻结 |
| DJI 无人机（Phase 2） | DJI Pilot / Mobile SDK / Cloud API / 媒体导入 | 不承诺 MVP 阶段飞控控制 |

MSkit 不替代无人机 Remote ID 合规义务；仅显示/记录无人机 Remote ID 状态。无人机本体、广播模块、飞手注册与飞行区域合规由运营方负责。

## 6. AI 性能与验收口径

| 指标 | 统一条件 | 目标 |
|------|----------|------|
| 模型推理 FPS | 640×640, batch=1, INT8, TensorRT/DeepStream, nvpmodel 最高档 | YOLOv8n ≥50 FPS |
| 端到端视频 FPS | 1080p 输入，含解码、resize、推理、跟踪、叠加、WebSocket 输出 | 单路 ≥25 FPS |
| 白天检测 mAP | 冻结测试集 ≥5,000 张 | mAP@0.5 ≥0.85 |
| 夜间/热成像 mAP | 冻结测试集 ≥3,000 张 | mAP@0.5 ≥0.70 |

所有指标须同时报告 Precision / Recall / F1 / 误报率 / 漏报率 / 端到端延迟。公开或早期 FPS 数据只作可行性参考，不作为验收依据。

YOLOv8/Ultralytics 不得默认进入闭源商业版本。项目必须在购买 Enterprise License 或替换为许可更宽松模型/训练框架之间二选一，并在 v1.4 冻结前完成性能复测。

## 7. 数据安全与导出

加密导出默认输出接收方公钥加密包，不输出明文。导出前需管理员 PIN / FIDO2 Key / 本地账户二次确认。导出包必须包含加密数据、SHA-256 哈希与签名清单。

## 9. 预算与商业口径

毛利率仅按硬件 BOM 计算，不含研发摊销、认证摊销、渠道费用、售后、物流、关税、装配、质检与坏件损耗。首批 50 套定义为 Pilot Batch，不作为完整回收研发成本的商业批次。研发摊销建议按 300–500 套周期核算。

研发预算按团队地区三档重估：极简创始团队 $400k–$500k；中国工程团队 $600k–$900k；美/欧/日团队 $1.2M–$2.0M。上述档位应含软件授权、数据集与审计工具，并另留 15% 应急金。

## 10. 外协执行口径

MSkit 外协采用“双层结构”：

- **一级外包包 A–F**：用于对外 SOW / RFP / 合同管理；
- **OP-01–OP-12**：作为内部二级 WBS、验收项、依赖项和风险跟踪项。

| 一级包 | 合并 OP | 主责 |
|--------|---------|------|
| A. Rugged Hardware & Mechatronics | OP-01 + OP-02 + OP-03 | 结构、电源、电池、热设计、环境预验证 |
| B. RF / Wireless / Certification | OP-04 + OP-11 中认证技术部分 | 地区化无线 SKU、天线、RF/EMC 预扫描、认证资料 |
| C. Edge Platform & Device Firmware | OP-05 + OP-09 + OP-12 | Jetson BSP、OTA、LoRa 低速遥测、RTK/GNSS/IMU 融合 |
| D. Application Software | OP-07 + OP-08 | 地图、导航、任务、报告、Dashboard、视频 UI 链路 |
| E. AI Data & Model Optimization | OP-06 | 数据集、训练、模型权重、TensorRT engine、模型层性能 |
| F. Manufacturing & Test Fixtures | OP-10 | 试产 SOP、工装、老化测试、出厂测试、良率报告 |
| 内部核心包 | 架构/安全/接口/验收/合规边界 | 不外包，负责最终整机签署 |

端到端视频 FPS、加密导出、整机 IP/EMC、Pilot Batch 出厂质量等跨包指标由内部 Integration Owner / QA 牵头联合验收，不能转嫁给单一外协方。
