# MSkit v1.3.2 MVP 场景工作流：搜救 + Hide and Seek

> **阶段**：M0 / MVP / 立项演示 / 融资路演  
> **用途**：按照具体任务场景推演组织方、搜索方、防护方工作流，并据此明确 MVP 架构、功能、数据对象和演示边界。  
> **优先级**：本文件是 MVP 场景工作流的权威源；MVP PLAN / CHECKLIST / DEMO PLAN 应引用本文件，不重复展开完整流程。  
> **合规边界**：仅用于民用搜救演练、实景游戏、赛事组织、安保训练和户外团队协作。不做无人机飞控控制、不自动下发航线、不做武器识别、不做作战控制。

---

## 1. MVP 场景取舍

MVP 不同时覆盖所有场景，而是优先完成两个脚本：

| 场景 | 参与方 | MVP 定位 | 优先级 |
|------|--------|----------|--------|
| 野外搜救 | 组织方 + 搜索方 | 展示社会价值和任务闭环；无防护方 | P0 |
| Hide and Seek 实景对抗赛 | 组织方 + 搜索方 + 防护方 | 展示完整三方叙事和游戏化商业价值 | P0 |

---

## 2. 通用 MVP 架构

```text
[Scenario Setup]
  - 活动区 / 搜索区 / 防护区 / 安全区 / 禁入区
  - 目标点 / 疑似点 / 诱饵 / 资源参数 / 胜负规则
        |
        v
[Data & Simulation Layer]
  - 卫星/离线地图
  - 无人机模拟或素材导入
  - 相机 footprint / 航迹 / 目标识别结果
        |
        v
[Detection & Evidence Layer]
  - 目标检测
  - 变化检测
  - 可疑点置信度 / 优先级 / 人工复核
        |
        v
[Planning Layer]
  - 搜索区更新
  - 路径规划
  - 覆盖规划
  - 重新规划触发器
        |
        v
[Tri-Party Dashboard]
  - 组织方态势 / 搜索方任务 / 防护方方案
        |
        v
[Report & Replay]
  - 任务报告
  - 事件时间线
  - 真实 vs 模拟披露
```

---

## 3. 通用数据对象

| 对象 | 说明 | MVP 必须实现 |
|------|------|--------------|
| Activity Area / 活动区 | 最大任务范围 | 是 |
| Search Area / 搜索区 | 搜索方重点搜索范围 | 是 |
| Safe Zone / 安全区 | 出发、撤离、补给、集结点，可多个 | 是 |
| No-Go Zone / 禁入区 | 障碍区、导调观察点、危险区、不可进入区 | 是 |
| Protection Area / 防护区 | Hide and Seek 中防护方阵地；搜救中通常不用 | Hide and Seek 必须 |
| Target Point / 目标点 | 真实目标位置，如遇险人员、游戏目标 | 是，组织方预置 |
| Suspect Point / 疑似点 | 检测得到或组织方预置的可疑点 | 是 |
| Decoy / 诱饵 | Hide and Seek 中防护方布设的干扰点 | Hide and Seek 必须 |
| Detection Event / 检测事件 | 一次目标检测或变化检测输出 | 是 |
| Priority / 优先级 | 可疑点优先级，基于置信度、距离、时效、人工修正 | 是 |
| Route / 路线 | 人员或小队路径建议 | 是 |
| UAV Sweep / 无人机扫描 | 无人机覆盖搜索区的航迹或模拟航迹 | 是，可模拟 |
| Report / 报告 | 任务结果、证据、路线、披露说明 | 是 |

# 场景一：野外搜救

## 4. 场景目标

野外搜救场景中没有防护方。MVP 的目标是展示：

1. 组织方基于先验信息划定活动区、搜索区和安全区。
2. 搜索方从安全区出发，使用无人机对搜索区进行拉网式检测。
3. 系统根据检测置信度生成可疑点和优先级。
4. 系统根据当前位置、道路/地形和可疑点优先级规划路线。
5. 搜索方接近并复核目标点，最终完成任务。

---

## 5. 组织方工作流：搜救

| 步骤 | 组织方动作 | 系统响应 | MVP 备注 |
|------|------------|----------|----------|
| O-SR-01 | 创建搜救任务 | 新建任务、时间线、地图项目 | 必须 |
| O-SR-02 | 划定大范围活动区 | 地图显示活动区边界 | 必须 |
| O-SR-03 | 根据先验信息标出重点搜索区 | 生成一个或多个搜索区 polygon | 必须 |
| O-SR-04 | 指定出发/撤离地点，即安全区 | 生成安全区，可多个 | 必须 |
| O-SR-05 | 预置目标点 | 在搜索区内随机或手工放置目标点，仅组织方可见 | MVP 特色功能 |
| O-SR-06 | 预置疑似点 | 放置不同形态的疑似标记，如石块 SOS 变体、衣物、反光物、动物/人形干扰 | MVP 特色功能 |
| O-SR-07 | 配置无人机/相机模拟参数 | 高度、速度、FOV、分辨率、航向、扫描间隔 | 可模拟 |
| O-SR-08 | 启动任务 | 搜索方视图只看到授权搜索区和安全区，不看到真实目标点 | 必须 |
| O-SR-09 | 查看进展 | 显示无人机覆盖、可疑点、搜索方位置、路线 | 必须 |
| O-SR-10 | 结束/复盘 | 生成报告：目标是否找到、耗时、路线、检测记录、人工修正 | 必须 |

---

## 6. 搜索方工作流：搜救

| 步骤 | 搜索方动作 | 系统响应 | MVP 备注 |
|------|------------|----------|----------|
| S-SR-01 | 到安全区 1 集结 | 记录搜索方起点 | 必须 |
| S-SR-02 | 放出无人机，扫描搜索区 | 执行 UAV Sweep；生成图像/视频帧和覆盖状态 | MVP 可模拟 |
| S-SR-03 | 预规划从安全区 1 到搜索区最近可到达点路径 | 系统生成初始路线 | 必须 |
| S-SR-04 | 接收无人机检测结果 | 生成可疑点、置信度、优先级、截图证据 | 必须 |
| S-SR-05 | 人工复核可疑点 | 可确认/驳回/提升/降低优先级 | 必须 |
| S-SR-06 | 更新搜索区 | 根据可疑点分布自动扩张/收缩/分裂搜索区，或人工修正 | 必须 |
| S-SR-07 | 重新规划路线 | 从当前位置到最高优先级可疑点，尽量经过其他高优先级可疑点 | 必须 |
| S-SR-08 | 开进至可疑点 | 更新当前位置、路线进度、事件时间线 | MVP 可模拟人员移动 |
| S-SR-09 | 现场复核 | 标记为目标点 / 非目标 / 待复核 | 必须 |
| S-SR-10 | 重复 S-SR-04 到 S-SR-09 | 直到满足完成条件或安全终止条件 | 必须 |

### 6.1 搜救完成条件

任一条件满足即可结束：

1. 搜索方到达真实目标点并确认。
2. 组织方确认目标点已被发现且证据充分。
3. 搜索时间耗尽。
4. 安全条件触发终止，如进入禁入区、天气/电量/演示异常。
5. 所有高优先级可疑点均已复核，但未发现目标，此时输出“未找到/需扩大搜索区”报告。

## 7. 搜救 MVP 特色功能

### 7.1 组织方预置目标点与疑似点

| 类型 | 设置者 | 搜索方是否可见 | 用途 |
|------|--------|----------------|------|
| 真实目标点 | 组织方 | 初始不可见 | 代表遇险人员所在地或关键目标 |
| 强疑似点 | 组织方或系统检测 | 检测后可见 | 例如白色石块 SOS、衣物、反光布、热源样例 |
| 弱疑似点 | 组织方或系统检测 | 检测后可见 | 形似目标但不完整，制造识别难度 |
| 干扰点 | 组织方 | 检测后可见 | 低置信度或误报演示 |

MVP 可实现“组织方随机布置”：在搜索区内按指定数量生成目标点、强疑似点、弱疑似点、干扰点。搜索方初始不可见，只有通过无人机扫描和检测事件才能发现。

### 7.2 路径规划模式

| 模式 | 名称 | 说明 | MVP 优先级 |
|------|------|------|------------|
| R1 | 道路优先 | 基于实际道路/步道/可通行线推荐路线 | P0 |
| R2 | 直线 + 禁入区绕行 | 缺少道路数据时，按直线接近并绕开禁入区 | P0 |
| R3 | 多可疑点串联 | 从当前位置到最高优先级点，并尽量经过其他高优先级点 | P0 |
| R4 | 人工路线校核 | 用户手工画路线，系统校核是否穿越禁入区、是否过长、是否偏离搜索区 | P0 |
| R5 | 地形/坡度推荐 | 基于 DEM/坡度/地表类型推荐徒步路线 | P1，可选 |
| R6 | 风险/时间/体力加权 | 在路径代价中加入距离、坡度、风险、优先级、剩余时间 | P1，可选 |
| R7 | 无人机覆盖优先 | 根据无人机扫描覆盖结果，引导人员前往覆盖盲区或高价值点 | P2 |

### 7.3 无人机航拍与检测模拟

MVP 不需要真机飞控控制，可以用以下方式生成“航拍/实拍”效果：

| 模式 | 说明 | 推荐度 |
|------|------|--------|
| D1 预录素材 | 使用真实无人机视频/图片 + 日志 | 最高，最稳定 |
| D2 地图裁切模拟 | 从卫星/离线地图按无人机高度、FOV、航迹裁切出帧，叠加目标/疑似点 | 推荐，适合可控演示 |
| D3 合成场景图 | 用组织方预置目标和背景图生成高清“航拍式”图像 | 推荐，适合 Hide and Seek/SOS 标记 |
| D4 实时采集 | HDMI/USB/RTSP 接入遥控器画面 | 可选，路演风险较高 |

模拟参数：高度、速度、航向、相机 FOV、分辨率、帧间隔、覆盖宽度、重叠率、光照/噪声、目标尺寸、遮挡系数。

### 7.4 可疑点优先级

建议使用 0–100 的优先级分数：

```text
Priority = 0.45 * DetectionConfidence
         + 0.20 * TargetSimilarity
         + 0.15 * SearchContextScore
         + 0.10 * RecencyScore
         + 0.10 * ManualAdjustment
```

| 因子 | 说明 |
|------|------|
| DetectionConfidence | 模型识别置信度 |
| TargetSimilarity | 与目标特征相似度，如 SOS、人体轮廓、衣物颜色、热源样式 |
| SearchContextScore | 是否在重点搜索区、是否靠近先验线索 |
| RecencyScore | 最近发现优先 |
| ManualAdjustment | 人工提升/降低 |

优先级等级：

| 分数 | 等级 | 处理建议 |
|------|------|----------|
| 80–100 | P0 | 立即规划前往 |
| 60–79 | P1 | 路线尽量经过 |
| 40–59 | P2 | 后续复核 |
| 0–39 | P3 | 低优先级或暂缓 |

### 7.5 搜索区更新规则

| 触发条件 | 自动动作 | 人工动作 |
|----------|----------|----------|
| 出现 P0/P1 可疑点聚类 | 将聚类周边扩展为高优先级搜索子区 | 组织方/搜索方可调整边界 |
| 某区域无人机覆盖完成且无发现 | 降低该区域优先级 | 可保留或标记“已搜索” |
| 搜索方确认误报 | 降低类似点权重 | 人工驳回并记录原因 |
| 搜索方确认目标相关线索 | 扩大周边搜索半径 | 组织方可新增搜索区 |
| 时间不足 | 收缩搜索区，只保留 P0/P1 和最近可达点 | 组织方确认 |

路径重新规划触发器：

1. 新增 P0 可疑点。
2. 最高优先级点被确认/驳回。
3. 搜索方当前位置变化超过阈值。
4. 禁入区/安全区被组织方修改。
5. 用户手动点击“重新规划”。
6. 剩余时间或无人机电量低于阈值。

# 场景二：Hide and Seek 实景对抗赛

## 8. 场景目标

Hide and Seek 场景中三方完整参与：

- **组织方**：导调、裁判、规则设置、资源设置、胜负判定。
- **搜索方**：使用无人机和检测能力搜索目标点。
- **防护方**：在规则允许范围内布设固定/机动目标、伪装、诱饵和覆盖/撤离方案。

MVP 的目标是展示一个完整回合：组织方建场 → 防护方布设 → 搜索方扫描 → 系统检测/更新 → 组织方裁判 → 报告。

---

## 9. 组织方工作流：Hide and Seek

| 步骤 | 组织方动作 | 系统响应 | MVP 备注 |
|------|------------|----------|----------|
| O-HS-01 | 创建游戏任务 | 新建活动和计时器 | 必须 |
| O-HS-02 | 划定游戏区域，即活动区 | 地图显示活动边界 | 必须 |
| O-HS-03 | 设定防护方阵地，即防护区 | 防护区必须在活动区内 | 必须 |
| O-HS-04 | 设定搜索方重点搜索区 | 搜索区通常大于且包含防护区 | 必须 |
| O-HS-05 | 设定安全区 | 搜索方出发/补给区，防护方撤离/补给区 | 必须 |
| O-HS-06 | 设定禁入区 | 障碍区、导调观察点、危险区 | 必须 |
| O-HS-07 | 设定搜索方资源 | 无人机高度、续航、换电时间、可见光/红外相机 FOV、焦距、分辨率 | MVP 可简化 |
| O-HS-08 | 设定防护方目标点 | 固定目标、机动目标；机动目标仅可在防护区内沿道路运动 | 必须 |
| O-HS-09 | 设定防护方资源 | 伪装器材、诱饵类型、数量、规格、布设时间 | MVP 可简化 |
| O-HS-10 | 设定胜负规则 | 规定时间内，搜索方发现目标数量达到阈值则胜，否则防护方胜 | 必须 |
| O-HS-11 | 启动回合 | 防护方进入布设阶段，搜索方进入准备阶段 | 必须 |
| O-HS-12 | 裁判判定 | 判断发现是否有效、目标是否到达阈值、是否越界 | MVP 可模拟 |
| O-HS-13 | 复盘报告 | 输出发现数量、误报、漏报、覆盖率、路线、胜负 | 必须 |

## 10. 搜索方工作流：Hide and Seek

| 步骤 | 搜索方动作 | 系统响应 | MVP 备注 |
|------|------------|----------|----------|
| S-HS-01 | 到安全区 1 集结 | 记录起点和开始时间 | 必须 |
| S-HS-02 | 查看搜索区、禁入区、资源 | 显示授权地图和资源限制 | 必须 |
| S-HS-03 | 设定无人机扫描航迹 | 用户选择拉网、分区、热点优先或系统建议 | 必须，可模拟 |
| S-HS-04 | 执行无人机扫描 | 生成图像/视频帧、覆盖状态、候选检测 | MVP 可模拟 |
| S-HS-05 | 接收目标检测/变化检测结果 | 生成可疑点、置信度、优先级 | 必须 |
| S-HS-06 | 人工复核可疑点 | 确认/驳回/备注/提升优先级 | 必须 |
| S-HS-07 | 标记疑似目标 | 生成“待裁判确认”的发现事件 | 必须 |
| S-HS-08 | 更新搜索区与无人机航迹 | 未覆盖区、热点区、疑似目标周边优先 | 必须 |
| S-HS-09 | 重复扫描与复核 | 直到时间耗尽或达到发现阈值 | 必须 |
| S-HS-10 | 提交结果 | 生成搜索方报告 | 必须 |

### 10.1 搜索方无人机航迹模式

| 模式 | 说明 | MVP 优先级 |
|------|------|------------|
| U1 拉网扫描 | 按平行航线覆盖搜索区 | P0 |
| U2 分区扫描 | 将搜索区分成网格，逐格扫描 | P0 |
| U3 热点优先 | 优先扫描已出现可疑点或先验线索区域 | P0 |
| U4 未覆盖优先 | 优先扫描覆盖盲区 | P1 |
| U5 电量约束 | 根据续航和返航时间切分任务 | P1，可选 |
| U6 传感器切换 | 可见光/红外视场差异演示 | P1，可选，MVP 可用模拟参数 |

## 11. 防护方工作流：Hide and Seek

| 步骤 | 防护方动作 | 系统响应 | MVP 备注 |
|------|------------|----------|----------|
| P-HS-01 | 查看防护区、安全区、禁入区和规则 | 只显示防护方授权视图 | 必须 |
| P-HS-02 | 选择目标点类型 | 固定目标或机动目标 | 必须 |
| P-HS-03 | 布设固定目标点 | 目标必须位于防护区内 | 必须 |
| P-HS-04 | 设定机动目标路线 | 机动目标只能在防护区内沿道路运动 | MVP 可模拟 |
| P-HS-05 | 布设诱饵 | 诱饵类型、数量、外观、热源/可见光特征 | 必须，可简化 |
| P-HS-06 | 布设伪装 | 设置目标可见性、遮挡、热特征衰减等参数 | MVP 可模拟 |
| P-HS-07 | 运行覆盖规划 | 评估哪些区域容易被无人机扫描发现，生成覆盖/隐藏建议 | 必须 |
| P-HS-08 | 规划撤离/补给路线 | 从目标区到安全区，避开禁入区 | 必须 |
| P-HS-09 | 提交防护方案 | 组织方可见，搜索方不可见 | 必须 |
| P-HS-10 | 回合中更新机动目标状态 | 按规则沿道路移动；组织方可查看 | 可选 |
| P-HS-11 | 回合结束复盘 | 输出防护成功/失败、暴露点、诱饵效果、覆盖缺口 | 必须 |

### 11.1 防护方资源模型

| 资源 | 参数 | MVP 作用 |
|------|------|----------|
| 固定目标 | 数量、尺寸、颜色/热特征、位置 | 作为真实目标 |
| 机动目标 | 数量、速度、道路约束、移动时间窗 | 高级演示，可模拟 |
| 诱饵 | 数量、外观、相似度、热特征 | 制造疑似点和误报 |
| 伪装 | 遮挡系数、颜色融合度、热衰减 | 影响检测置信度 |
| 安全区 | 撤离/补给位置 | 防护方路径规划 |
| 禁入区 | 不可进入范围 | 校核防护路线合法性 |

### 11.2 覆盖规划 / 防护规划输出

| 输出 | 说明 |
|------|------|
| 防护区地图 | 目标、诱饵、安全区、禁入区 |
| 目标暴露风险 | 基于无人机高度、FOV、覆盖航迹和伪装参数估计 |
| 覆盖盲区 | 搜索方当前航迹难以覆盖的区域 |
| 诱饵有效性 | 诱饵与真实目标的相似度和可能误导概率 |
| 撤离路线 | 从目标点/防护区到安全区的路线 |
| 防护报告 | 提交给组织方，搜索方不可见 |

## 12. Hide and Seek 胜负规则

| 条件 | 判定 |
|------|------|
| 搜索方在规定时间内发现并确认目标点数量 ≥ 阈值 | 搜索方胜，防护方负 |
| 搜索方在规定时间内发现目标点数量 < 阈值 | 防护方胜，搜索方负 |
| 搜索方进入禁入区 | 可扣分或判违规 |
| 防护方目标点超出防护区 | 防护方违规，目标无效或扣分 |
| 机动目标离开道路/允许范围 | 防护方违规 |
| 检测结果未被组织方裁判确认 | 不计入有效发现 |
| 时间耗尽 | 自动结算 |

MVP 可先实现固定规则：

```text
TimeLimit = 15 min
TargetCount = 3
WinThreshold = 2
ValidDiscovery = DetectionEvent confirmed by Organizer
```

## 13. 搜索区 / 防护区 / 路线更新机制

### 13.1 搜索方更新

| 触发 | 动作 |
|------|------|
| 新增高置信度可疑点 | 将该点周边设为热点搜索子区 |
| 发现诱饵或误报 | 降低同类特征权重 |
| 某区域已被无人机覆盖 | 标记为已覆盖，降低优先级 |
| 时间不足 | 收缩到 P0/P1 可疑点和最近可达路线 |
| 找到有效目标 | 重新规划到下一高优先级点 |

### 13.2 防护方更新

| 触发 | 动作 |
|------|------|
| 搜索方扫描覆盖扩大 | 重新评估目标暴露风险 |
| 诱饵被发现 | 记录诱饵效果，调整评分 |
| 机动目标移动 | 更新目标可能区域 |
| 搜索方接近目标区 | 触发防护方风险提示，MVP 可仅在组织方/防护方视图显示 |

### 13.3 组织方裁判更新

| 触发 | 动作 |
|------|------|
| 搜索方提交发现 | 裁判确认有效/无效 |
| 搜索方或防护方越界 | 标记违规 |
| 时间耗尽 | 结算 |
| 目标达到阈值 | 提前结束回合 |

## 16. 真实 vs 模拟披露

MVP 必须在演示脚本和报告中披露哪些能力是真实运行，哪些是模拟/预置：

| 能力 | MVP 披露口径 |
|------|--------------|
| 地图显示 | 真实离线地图或静态地图素材 |
| 无人机视频/图片 | 可为真实预录素材或模拟生成素材 |
| 无人机飞行 | 不控制真机；航迹可模拟或由日志导入 |
| 目标检测 | 可真实运行模型，也可用预计算结果兜底；必须披露 |
| 变化检测 | 可为简化算法或预置样例；必须披露 |
| 路径规划 | MVP 真实运行简化规划算法 |
| 地形/坡度路径 | 若未接 DEM，必须标注为未来能力 |
| 搜索区更新 | 可规则驱动 + 人工修正 |
| 覆盖规划 | MVP 可用简化几何覆盖模型 |
| 机动目标 | 可模拟，不代表真实追踪能力 |
| 热成像 | 可用样例素材，不代表已集成正式热成像硬件 |
