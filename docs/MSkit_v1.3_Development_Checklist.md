# MSkit v1.3 开发Checklist

> 本Checklist按模块和开发阶段组织，供开发团队逐项核对。每个条目包含：□ 待完成 / ☑ 已完成 / ⏳ 进行中
> 最后更新：2026年7月
> 相关文档：[主规格 MSkit_v1.3_Spec.md](MSkit_v1.3_Spec.md) ｜ [可验收开发计划](MSkit_v1.3_Development_Plan.md) ｜ [BOM 成本表](MSkit_v1.3_BOM_Cost_Table.md)
> 阶段划分与里程碑（M1–M6、18 个月）以主规格 §9 实施计划为准。
> 本 Checklist 为逐项勾选清单；每项的**负责人、量化验收标准（DoD）、关口绑定**见[开发计划](MSkit_v1.3_Development_Plan.md)。

## Phase 1: 需求与架构 (M1-M2)

### 1.1 需求冻结
- [ ] v1.3 Spec文档签署（项目经理+技术负责人+法务）
- [ ] 用例场景确认（5个目标场景，每个至少3个用户故事）
- [ ] 性能KPI基线确认（推理FPS、定位精度、续航时间）
- [ ] 合规边界审核（禁用功能清单、术语审查）

### 1.2 系统架构设计
- [ ] 硬件架构图（模块连接关系、电源树、信号完整性）
- [ ] 软件架构图（容器拓扑、服务依赖、数据流）
- [ ] 接口定义文档（FastAPI schema v1.0、Protobuf定义、MQTT topic规范）
- [ ] 数据库schema设计（InfluxDB retention policy、PostgreSQL表结构）

### 1.3 技术选型验证
- [ ] Jetson Orin Nano Super + JetPack 6.2.1 POC验证
- [ ] YOLOv8n INT8推理性能基准测试（目标：>50 FPS）
- [ ] 热成像模块（FLIR Boson 640）数据接口验证（USB/CSI-2）
- [ ] u-blox ZED-F9P RTK定位精度验证（<2cm水平误差）
- [ ] LoRa Mesh（E22-900T30D）通信距离测试（>3km视距）
- [ ] 10.1"高亮显示屏阳光下可视性验证（>1000nits）

### 1.4 团队组建
- [ ] 嵌入式软件工程师（2名）JD发布+面试
- [ ] AI/算法工程师（1名）JD发布+面试
- [ ] 硬件工程师（1名，可选外包）签约
- [ ] 结构/外壳设计（外包）RFQ发布+供应商选定
- [ ] 开发环境搭建（Jetson开发套件×3、测试设备采购）

## Phase 3: 软件开发 (M3-M10)

### 3.1 BSP & JetPack定制
- [ ] JetPack 6.2.1 基础镜像构建
- [ ] 内核配置裁剪（移除无用驱动，减少启动时间）
- [ ] Docker + Docker Compose环境配置
- [ ] OTA更新机制设计（基于Mender/Rauc）
- [ ] nvpmodel功耗模式配置（7W/15W/25W三档）

### 3.2 MAP-NAV模块
- [ ] MapLibre GL离线瓦片服务搭建
- [ ] PostGIS数据库+离线OSM数据导入
- [ ] RTK定位数据接入（NMEA→坐标转换）
- [ ] 离线路径规划算法（A*/Dijkstra）
- [ ] 地图加载性能优化（目标：<5秒加载）
- [ ] 地形高程数据集成（SRTM/DEM）

### 3.3 SENSE模块
- [ ] YOLOv8n INT8 TensorRT引擎生成
- [ ] 人员/动物检测模型训练（自定义数据集）
- [ ] 热成像目标检测pipeline（双光谱融合）
- [ ] 检测性能基准测试（白天>92% mAP，夜间>75% mAP）
- [ ] DeepStream pipeline搭建（多路视频流）
- [ ] 推理FPS优化（目标：>50 FPS@YOLOv8n INT8）

### 3.4 TASK/SIM模块
- [ ] 任务管理数据模型设计
- [ ] 任务分配算法（基于位置+能力约束）
- [ ] Milsim场景推演引擎（简化版红蓝对抗）
- [ ] 时序数据记录与回放
- [ ] 任务报告自动生成（Jinja2模板）

### 3.5 MESH通信协议栈
- [ ] LoRa Mesh网络层协议实现
- [ ] 节点发现与路由维护
- [ ] 位置/状态数据广播（周期+触发）
- [ ] 消息加密（AES-256-GCM）
- [ ] 网络分区自愈机制
- [ ] 8节点组网压力测试

### 3.6 DASH/ALERT/REPORT模块
- [ ] 实时仪表盘Web UI（React/Vue）
- [ ] 告警规则引擎（阈值/地理围栏/AI检测触发）
- [ ] 告警通知（语音/LED/屏幕/LoRa广播）
- [ ] 报告模板系统（PDF/Word导出）
- [ ] 报告哈希验证（SHA-256）

### 3.7 安全与加密
- [ ] LUKS全盘加密配置
- [ ] TPM 2.0（SLB9670）集成与密钥密封
- [ ] Secure Boot启用
- [ ] 访问控制（RBAC，基于角色的权限）
- [ ] 审计日志系统（保留1年）
- [ ] SBOM生成（SPDX格式）
