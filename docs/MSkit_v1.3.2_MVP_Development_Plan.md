# MSkit v1.3.2 MVP 开发计划（三方路演版）

> **阶段**：M0 / 立项演示 / 融资路演  
> **周期**：6 周  
> **目标**：用最小硬件和最小软件闭环，验证 MSkit 的三方叙事：黑方 PNT/规划、红方 GOD/COD、蓝方 CCD。  
> **主脚本**：Hide and Seek 实景对抗赛。  
> **备选脚本**：野外搜救。  
> **边界**：MVP 不验证 Jetson、加固箱、LoRa、RTK、IP/EMC/认证和量产 BOM；不做无人机飞控控制、不下发航线、不做自动起降。

---

## 1. MVP 成功标准

MVP 成功不是“做完正式产品”，而是证明以下 5 件事：

1. 黑 / 红 / 蓝三方叙事能被 3 分钟讲清楚。
2. Hide and Seek 主脚本能完整跑通：黑方建场、红方发现、蓝方覆盖、黑方裁判/报告。
3. 野外搜救备选脚本能跑通核心流程：黑方划区、红方搜索、PNT 重规划、报告导出。
4. GOD/COD、CCD、PNT/路径规划都能在笔记本上稳定演示。
5. 全程不展示飞控控制、武器识别、打击排序、作战控制等高风险能力。

## 2. MVP 范围

### 2.1 必须做

| 模块 | 方别 | MVP 功能 | 验收口径 |
|------|------|----------|----------|
| 三方角色 | 黑/红/蓝 | 黑方、红方、蓝方视图或过滤器 | UI 中可切换或明确展示三方数据层 |
| 活动配置 | 黑方 | 创建活动区、红方搜索区、蓝方保护/隐藏区、禁入区 | 可保存并显示在地图上 |
| PNT / 时间线 | 黑方 | 导入无人机日志或模拟轨迹，显示轨迹、事件点、时间线 | 轨迹与事件点可联动 |
| GOD | 红方 | 从视频/图片中检测民用目标 | 至少 1–3 类目标可稳定演示 |
| COD | 红方/蓝方 | 对比基线与当前素材，显示变化点 | 至少 2 组样例可演示变化高亮 |
| CCD | 蓝方 | 设计保护区、观察点、覆盖路线、盲区提示 | 可生成覆盖/采集方案 |
| 路径规划 | 黑方/红方/蓝方 | 生成绕开禁入区的搜索/巡查路径 | 地图上可显示路径线和说明 |
| 报告导出 | 黑方 | 导出黑方总览、红方发现、蓝方方案、路径建议 | HTML/Markdown 优先，PDF 可选 |
| 演示脚本 | 黑方主叙事 | 3 分钟融资版 + 8–10 分钟技术版 | 录屏备份完成 |

### 2.2 可以模拟

| 模块 | 模拟方式 | 说明 |
|------|----------|------|
| 实时视频 | 预录视频循环播放 | 降低路演风险 |
| 无人机轨迹 | GPX/SRT/CSV 或手工轨迹 | MVP 不验证真实 RTK |
| 热成像 | 样例热成像素材 | 不采购正式热成像模组 |
| 多节点 | 虚拟红/蓝节点 | 不做 LoRa 实物组网 |
| 裁判计分 | 手工/固定规则 | 不做复杂赛事系统 |

### 2.3 明确不做

- 不做无人机飞控控制。
- 不自动起飞、自动降落、自动航线下发。
- 不做 Jetson / 加固箱 / LoRa / RTK 实物集成。
- 不做 IP65 / MIL-STD-810H / EMC / SRRC / FCC / CE 实测。
- 不承诺真实 RTK <2cm 精度。
- 不做武器识别、军事化目标分类、打击排序、作战控制。
- 不把 MVP 设备成本混入正式产品 BOM。

```text
[Drone / RC / Media Files]
        |
        | video / image / logs
        v
[Ingest Service]
        |
        +--> [Media Store]
        +--> [Telemetry / PNT Timeline]
        |
        v
[AI Pipeline: Red GOD / COD]
        |
        v

        |
        +--> [Blue CCD Planner]
        +--> [Black Route / PNT Planner]
        |
        v

        |
        v
[Demo Report Export]
```

## 4. 技术选型

| 层级 | MVP 选型 | 说明 |
|------|----------|------|
| 前端 | React / Vite / MapLibre GL | 三方视图、地图、路径、报告预览 |
| 后端 | Python FastAPI | 数据接入、AI 调用、规划接口、报告生成 |
| 数据库 | SQLite | 单机演示足够，后续再迁移 PostgreSQL |
| 文件存储 | 本地目录 | 视频、图片、日志、报告 |
| AI | YOLO 系列或替代检测模型 | MVP 优先演示稳定性；对外展示前确认许可证 |
| COD | 简化帧差/特征对比/人工基线 | 不追求正式算法指标 |
| CCD | 简化覆盖路径 + 观察点生成 | 优先可解释、可视化 |
| 路径规划 | A* / Dijkstra / 简化网格路径 | 支持禁入区绕行 |
| 报告 | HTML / Markdown，PDF 可选 | 路演先保证导出稳定 |
| 部署 | 本机运行或 Docker Compose | 优先减少环境依赖 |

## 5. 6 周计划

| 周期 | 目标 | 交付物 | Go/No-Go |
|------|------|--------|----------|
| W1 | 三方叙事与素材冻结 | 黑/红/蓝角色、Hide and Seek 主脚本、野外搜救备选脚本、样例素材、UI 线框 | 场景、素材、合规边界确认 |
| W2 | 数据接入 + 黑方地图/PNT | 视频/图片/日志导入、离线地图、轨迹上图、黑方时间线 | 能展示“视频 + 轨迹 + 地图 + 黑方全局视图” |
| W3 | 红方 GOD/COD | 检测框、类别、置信度、发现点上图、变化高亮 | 至少 1 类目标稳定检测；COD 可演示变化 |
| W4 | 蓝方 CCD | 保护区/禁入区绘制、覆盖路线、盲区提示、巡查路径 | 能演示蓝方 CCD 方案 |
| W5 | 三方联动 + 报告 | 黑方规划、红方发现、蓝方覆盖叠加；报告导出 | Hide and Seek 主脚本可完整跑通 |
| W6 | 路演打磨 | Demo 数据固化、异常兜底、录屏备份、3 分钟/10 分钟脚本 | 连续演示 10 分钟无崩溃 |
