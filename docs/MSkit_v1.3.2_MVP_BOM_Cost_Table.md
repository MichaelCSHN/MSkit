# MSkit v1.3.2 MVP BOM / 演示成本表（三方路演版）

> **阶段**：M0 / 立项演示 / 融资路演  
> **用途**：估算 MVP 演示所需设备、软件、素材和短期外协成本。  
> **边界**：本表不是正式产品 BOM，不用于 Pilot Batch 或量产定价；正式硬件 BOM 仍见 `MSkit_v1.3.2_BOM_Cost_Table.md`。

---

## 1. MVP 成本口径

| 口径 | 说明 |
|------|------|
| 演示设备成本 | 笔记本、无人机、遥控器、采集卡、外置硬盘、演示显示设备等 |
| 软件开发成本 | 6 周 MVP Web App、目标检测/变化检测、覆盖规划、定位态势、报告导出开发投入 |
| 数据与素材成本 | 无人机视频、图片、日志、地图瓦片、样例场景、录屏素材 |
| 短期外协成本 | 2–6 周短 Sprint，不绑定长期正式工程 SOW |
| 不计入项 | Jetson、加固箱、LoRa、RTK、热成像正式模组、IP/EMC/认证、工装、Pilot Batch |

| HDMI/USB 采集卡 | 1080p 采集 | 1 | $50–$200 | $50–$200 | 可选，实时画面采集 |

## 3. MVP 软件 / 数据 / 外协成本

| 成本项 | 建议投入 | 估算成本 | 说明 |
|--------|----------|----------|------|
| Product / Demo 设计 | 1 人 × 2–6 周 | 内部或 $3,000–$12,000 | 三方叙事、脚本、路演结构 |
| Full-stack MVP | 1 人 × 4–6 周 | $8,000–$24,000 | FastAPI、React、地图、报告、三方视图 |
| AI MVP | 1 人 × 3–6 周 | $6,000–$24,000 | 目标检测/变化检测 模型接入、样例优化、输出格式 |
| GIS / Planning | 0.5–1 人 × 2–4 周 | $3,000–$12,000 | 定位态势、路径规划、覆盖规划、离线地图 |
| UI / Deck 设计 | 0.5 人 × 1–3 周 | $1,500–$6,000 | 路演 UI、截图、演示素材、deck 素材 |
| QA / Demo Ops | 0.5 人 × 1–2 周 | $1,500–$5,000 | 稳定性、演示机、录屏、兜底脚本 |
| 法务/许可证审查 | 外部顾问或内部 | $1,000–$5,000 | 模型、地图、素材、无人机合规措辞 |
| 素材采集 | 半天–2 天 | $0–$3,000 | Hide and Seek / 野外搜救样例素材 |

**MVP 软件/服务小计**：约 **$24,000–$91,000**。极简内部团队可压缩到 **$10,000–$25,000** 的现金支出，但需要创始团队亲自开发。

## 4. MVP 总预算建议

| 档位 | 适用情况 | 现金预算 | 特点 |
|------|----------|----------|------|
| 极简版 | 已有笔记本/无人机，创始团队自研 | $5,000–$15,000 | 适合内部立项验证，但 UI 和稳定性可能一般 |
| 标准路演版 | 采购或租用设备，小团队 6 周开发 | $25,000–$60,000 | 推荐档位，能支撑融资路演和早期客户访谈 |
| 强演示版 | 更好 UI、专业素材、录屏、双脚本、设计支持 | $60,000–$120,000 | 适合正式 Demo Day、投资人密集路演、客户试点前展示 |

推荐当前采用：**标准路演版**。

## 5. 三方功能与成本归属

| 三方能力 | MVP 成本归属 | 说明 |
|----------|--------------|------|
| 组织方 定位态势 / 全局规划 | Full-stack + GIS | 地图、轨迹、时间线、路径规划、报告 |
| 搜索方 目标检测 / 变化检测 | AI + 素材采集 | 检测、变化识别、候选点、复核 |
| 防护方 覆盖规划 | Full-stack + GIS | 保护区、禁入区、覆盖路线、盲区提示 |
| 三方视图/权限 | Full-stack | MVP 可简化为视图过滤，不做完整 RBAC |
| 报告导出 | Full-stack + UI | HTML/Markdown 优先，PDF 可选 |
| 合规措辞 | PM + LEG | 对外脚本和 UI 禁用高风险表述 |

## 7. 不建议采购 / 不纳入 MVP

| 项目 | 原因 |
|------|------|
| Jetson Orin 开发板 | MVP 可先用笔记本验证软件闭环 |
| Pelican 加固箱 | MVP 不验证结构/防护 |
| LoRa Mesh 模块 | MVP 用虚拟节点或日志模拟 |
| RTK 基站/流动站 | MVP 不承诺 RTK 精度 |
| FLIR Boson 正式模组 | 成本高，MVP 可用已有热成像素材 |
| 大容量电池/BMS | MVP 不验证续航和热插拔 |
| EMC/IP/环境测试 | 属正式工程样机阶段 |
| 工装治具 | 属 Pilot Batch / 量产阶段 |

---

## 8. MVP 后转正式 BOM 的触发条件

满足以下条件后，再进入正式 BOM 采购和工程样机阶段：

1. Hide and Seek 主脚本经至少 3 次外部演示验证可理解。
2. 野外搜救备选脚本能证明非游戏场景价值。
3. 投资人或潜在客户明确认可三方叙事。
4. 目标检测/变化检测、覆盖规划、定位态势/路径规划三条功能线都有明确产品优先级。
5. 目标首批客户场景明确：赛事、Milsim、安保训练、搜救或巡检。
6. 才启动 Jetson、加固箱、RTK、LoRa、热成像和电池系统的正式样机 BOM。
