# MSkit v1.3 开发计划（可验收版）

> **基线**：[MSkit_v1.3_Spec.md](MSkit_v1.3_Spec.md)（主规格）
> **周期**：18 个月（M1–M18，M = 项目月）｜**团队**：冷启动，4 全职 + 2 兼职/外包
> **配套**：[开发 Checklist](MSkit_v1.3_Development_Checklist.md)（逐项勾选）｜[BOM 成本表](MSkit_v1.3_BOM_Cost_Table.md)
> **目的**：把主规格的目标拆解为**具体、可指派、带可量化验收标准（DoD）**的工作包（WP），并绑定 Go/No-Go 关口。

## 0. 编制约定

### 0.1 工作包（WP）字段

每个工作包含以下字段，"完成"必须**同时**满足验收标准与评审签署：

| 字段 | 含义 |
|------|------|
| WP-ID | 唯一编号，`WP-<阶段>.<序号>` |
| 交付物 | 可入库的产物（代码/文档/样机/报告/证书） |
| 负责 | 主责角色（见 0.2），首个为 Owner |
| 起止 | 项目月区间 |
| 依赖 | 前置 WP-ID |
| 验收标准（DoD） | **可量化、可复现**的通过条件 |
| 验证方法 | 谁用什么手段判定通过 |

### 0.2 角色

| 代号 | 角色 | 配置 |
|------|------|------|
| PM | 项目经理 | 1（兼产品） |
| EMB | 嵌入式软件工程师 | 2（全职） |
| AI | AI/算法工程师 | 1（全职） |
| FS | 全栈工程师 | 1（全职） |
| HW | 硬件工程师 | 外包/兼职 |
| ME | 结构/外壳设计 | 外包 |
| QA | 测试 | 团队兼（EMB/FS 轮值） |
| LEG | 法务/合规 | 外部顾问 |
| SCM | 供应链/采购 | PM 兼 |

### 0.3 优先级与状态

- **优先级**：`P0` 关键路径/阻塞里程碑 ｜ `P1` 重要 ｜ `P2` 次要可延后
- **状态**：□ 待开始 ｜ ⏳ 进行中 ｜ ☑ 已验收（DoD 全通过 + 签署）
- **DoD 通过原则**：所有量化阈值**实测达标**（非估算）、产物已入库（git/文档库）、有可追溯的验证记录（报告/日志/截图）。

### 0.4 度量口径（避免验收争议）

| 指标 | 统一测量条件 |
|------|--------------|
| AI 推理 FPS | DeepStream，batch=1，1080p 输入，INT8，nvpmodel 最高档，稳态 5 分钟均值 |
| mAP | mAP@0.5，MSkit 自建验证集（见 WP-3.3.1），非 COCO 泛化值 |
| RTK 精度 | 静态观测 1 小时，相对已知基准点的水平/垂直 StdDev |
| 续航 | 标准负载循环（定义见 WP-2.4.3），放电至 BMS 保护截止 |
| 告警响应 | 触发源事件时间戳 → 通知落地时间戳，本地端到端 |
| 温度/防水/振动/冲击 | 按 §5.1 对应 MIL-STD-810H / IEC 方法，第三方或自建箱体 |

## 11. 运作机制

- **节奏**：每 Phase kickoff 定范围；每周更新 Checklist 状态；每 Phase 末 Go/No-Go。
- **升级**：单项阻塞 >3 天未解，立即进风险登记册并升级 PM。
- **变更**：影响 KPI/BOM/接口的变更走 CCB。
- **可追溯**：每条 DoD 的验证记录（报告/日志/截图）入文档库，关口评审可回溯。
