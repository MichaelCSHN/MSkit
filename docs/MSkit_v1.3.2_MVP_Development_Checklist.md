# MSkit v1.3.2 MVP 开发 Checklist（三方路演版）

> **阶段**：M0 / 立项演示 / 融资路演  
> **用途**：用于 6 周 MVP 的开发、验收和路演前检查。  
> **主脚本**：Hide and Seek 实景对抗赛。  
> **备选脚本**：野外搜救。  
> **边界**：本 Checklist 优先级高于正式工程 Checklist；正式 `MSkit_v1.3.2_Development_Checklist.md` 用于 M1 之后的工程样机与 Pilot Batch。

---

## 0. MVP 启动前检查

- [ ] 项目定位已冻结：MSkit = 黑 / 红 / 蓝三方智能规控平台。
- [ ] 主脚本已冻结：Hide and Seek 实景对抗赛。
- [ ] 备选脚本已冻结：野外搜救。
- [ ] 三方角色已冻结：黑方=组织者/裁判/调度，红方=搜索/检测小队，蓝方=防护/值守小队。
- [ ] 对外合规边界已冻结：不飞控、不自动航线、不武器识别、不打击排序、不作战控制。
- [ ] MVP 硬件方案已选：方案 A（笔记本 + 遥控器 + 无人机）或方案 B（笔记本 + 离线素材）。
- [ ] MVP 预算档位已选：极简版 / 标准路演版 / 强演示版。
- [ ] 6 周计划已确认。
- [ ] 演示负责人、技术负责人、AI 负责人、GIS/规划负责人、QA 负责人已明确。

---

- [ ] 黑方 3 分钟讲解词完成。
- [ ] 红方 GOD/COD 讲解词完成。
- [ ] 蓝方 CCD 讲解词完成。
- [ ] Hide and Seek 主脚本流程完成。
- [ ] 野外搜救备选脚本流程完成。

- [ ] 野外搜救样例视频/图片/日志准备完成。

## 2. W2：数据接入 + 黑方 PNT

- [ ] 项目创建 / 加载功能可用。
- [ ] 活动区域可绘制。
- [ ] 红方搜索区可绘制。
- [ ] 蓝方保护/隐藏区可绘制。
- [ ] 禁入区 / 安全区可绘制。
- [ ] 视频文件可导入。
- [ ] 图片文件可导入。
- [ ] 飞行日志可导入，至少支持 GPX/SRT/CSV 中一种。
- [ ] 离线地图可加载，目标 <5 秒。
- [ ] 无人机轨迹可显示在地图上。
- [ ] 事件点可显示在地图上。
- [ ] 黑方 PNT 时间线初版可用。

**W2 Go/No-Go**：能展示“视频 + 轨迹 + 地图 + 黑方全局视图”。

## 3. W3：红方 GOD/COD

- [ ] GOD 模型可对样例视频或图片运行。
- [ ] 检测框、类别、置信度可显示。
- [ ] 至少 1 类民用目标稳定检测。
- [ ] 检测结果可落到地图或事件列表。
- [ ] 候选目标列表可显示。
- [ ] 候选目标可确认 / 驳回 / 备注。
- [ ] COD 可对至少 2 组样例素材做变化高亮。
- [ ] 红方搜索轨迹可显示。
- [ ] 红方发现点可进入报告。
- [ ] 模型、权重、样例数据许可证状态已记录。

**W3 Go/No-Go**：红方 GOD/COD 可以产生可解释发现。

---

## 4. W4：蓝方 CCD

- [ ] 蓝方保护区可绘制和编辑。
- [ ] 蓝方禁入区 / 缓冲区可绘制。
- [ ] 起降点 / 观察点可设置。
- [ ] CCD 可生成覆盖/采集建议。
- [ ] 覆盖路线可显示在地图上。
- [ ] 盲区 / 未覆盖区可显示。
- [ ] 巡查路径可生成。
- [ ] CCD 可引用红方 COD 变化点作为输入或提示。
- [ ] 蓝方方案可进入报告。

**W4 Go/No-Go**：蓝方 CCD 方案可以被非技术观众理解。

- [ ] 报告包含：地图截图、红方发现、蓝方方案、路径建议、合规说明。
- [ ] Hide and Seek 主脚本完整跑通。
- [ ] 野外搜救备选脚本核心流程跑通。

**W5 Go/No-Go**：三方闭环和报告导出可完整演示。

## 7. 路演当天检查

- [ ] 笔记本电源和电池正常。
- [ ] 外接显示 / 投影测试通过。
- [ ] 鼠标、转接头、HDMI/USB-C 线齐备。
- [ ] 外置 SSD / 备份 U 盘可读取。
- [ ] 无人机素材本地备份可用。
- [ ] 如现场飞行，无人机、电池、遥控器、存储卡检查完成。
- [ ] 如不现场飞行，离线素材和录屏备份可用。

### 7.2 软件检查

- [ ] 本地服务可启动。
- [ ] 前端页面可打开。
- [ ] 地图可加载。
- [ ] 演示项目可加载。
- [ ] 视频 / 图片 / 日志可读取。
- [ ] GOD/COD 结果可显示。
- [ ] CCD 方案可显示。
- [ ] 路径规划可显示。
- [ ] 报告可导出。

### 7.3 叙事检查

- [ ] 先讲场景，再讲产品，不先讲硬件。
- [ ] 先讲黑/红/蓝，再讲 GOD/COD/CCD/PNT。
- [ ] Hide and Seek 主脚本优先展示。
- [ ] 野外搜救作为社会价值备选展示。
- [ ] 不使用军事化或飞控控制表述。

## 9. 转入正式工程的条件

满足以下条件后，才进入正式工程 PLAN / BOM / CHECKLIST：

- [ ] MVP 完成至少 3 次外部演示。
- [ ] 至少 2 类目标用户认可三方叙事。
- [ ] 明确首个商业主场景：Hide and Seek / Milsim / 安保训练 / 野外搜救 / 巡检之一。
- [ ] 明确正式硬件形态：Backpack / Team / Field 的优先级。
- [ ] 明确是否需要热成像、RTK、LoRa、加固箱。
- [ ] 明确预算和融资节点。
- [ ] 通过 CCB 决定进入 M1 Spec 冻结。
