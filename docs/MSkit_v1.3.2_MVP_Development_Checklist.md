# MSkit v1.3.2 MVP 开发 Checklist（三方路演版）

> **阶段**：M0 / 立项演示 / 融资路演  
> **用途**：用于 6 周 MVP 的开发、验收和路演前检查。  
> **主脚本**：Hide and Seek 实景对抗赛。  
> **备选脚本**：野外搜救。  
> **场景工作流权威源**：`MSkit_v1.3.2_MVP_Scenario_Workflows.md`。  
> **边界**：本 Checklist 优先级高于正式工程 Checklist；正式 `MSkit_v1.3.2_Development_Checklist.md` 用于 M1 之后的工程样机与 Pilot Batch。

---

## 0. MVP 启动前检查

- [ ] 项目定位已冻结：MSkit = 组织 / 搜索 / 防护三方智能规控平台。
- [ ] 场景工作流文档已冻结：搜救 + Hide and Seek。
- [ ] 主脚本已冻结：Hide and Seek 实景对抗赛。
- [ ] 备选脚本已冻结：野外搜救。
- [ ] 三方角色已冻结：组织方=组织者/裁判/调度，搜索方=搜索/检测小队，防护方=防护/值守小队。
- [ ] 搜救场景明确不使用防护方。
- [ ] 对外合规边界已冻结：不飞控、不自动航线、不武器识别、不打击排序、不作战控制。
- [ ] 真实 vs 模拟披露口径已冻结：脚本/页面标注哪些实时真实、哪些模拟/预置（见 MVP_Demo_Plan §10）；不得把模拟当正式性能。
- [ ] MVP 硬件方案已选：方案 A（笔记本 + 遥控器 + 无人机）或方案 B（笔记本 + 离线素材）。
- [ ] MVP 预算档位已选：极简版 / 标准路演版 / 强演示版。
- [ ] 6 周计划已确认。
- [ ] 演示负责人、技术负责人、AI 负责人、GIS/规划负责人、QA 负责人已明确。

## 3. W3：搜索方目标检测/变化检测 + 可疑点管理

- [ ] 目标检测模型可对样例视频或图片运行。
- [ ] 检测框、类别、置信度可显示。
- [ ] 至少 1 类民用目标稳定检测。
- [ ] 检测结果可落到地图或事件列表。
- [ ] 候选目标列表可显示。
- [ ] 可疑点置信度可显示。
- [ ] 可疑点优先级可自动生成。
- [ ] 可疑点可确认 / 驳回 / 备注 / 调整优先级。
- [ ] 优先级等级 P0/P1/P2/P3 可显示。
- [ ] 变化检测可对至少 2 组样例素材做变化高亮。
- [ ] 搜索方搜索轨迹可显示。
- [ ] 搜索方发现点可进入报告。
- [ ] 模型、权重、样例数据许可证状态已记录。

**W3 Go/No-Go**：搜索方目标检测/变化检测可以产生可解释发现，并形成可疑点优先级。

## 4. W4：搜救路径规划 + Hide and Seek 覆盖规划

### 4.1 搜救路径规划

- [ ] 支持从安全区到搜索区最近可到达点的初始路径。
- [ ] 支持从当前位置到最高优先级可疑点的路径。
- [ ] 支持尽量经过多个高优先级可疑点的多点串联路径。
- [ ] 支持禁入区绕行。
- [ ] 支持人工指定/修改路线。
- [ ] 支持人工路线可通行性校核：是否穿越禁入区、是否明显过长、是否偏离搜索区。
- [ ] 道路优先模式或直线绕行模式至少实现一种。
- [ ] 地形/坡度模式如未实现，必须标注为未来能力。

### 4.2 搜索区更新

- [ ] 新增高优先级可疑点时，可生成热点搜索子区。
- [ ] 已覆盖且无发现区域可标记为已搜索。
- [ ] 确认误报后可降低类似点权重或标记为已驳回。
- [ ] 发现目标相关线索后可扩展周边搜索区。
- [ ] 支持人工修正搜索区。
- [ ] 支持手动触发重新规划。

### 4.3 Hide and Seek 覆盖规划

- [ ] 防护区可绘制和编辑。
- [ ] 防护方禁入区 / 缓冲区可绘制。
- [ ] 固定目标点可布设在防护区内。
- [ ] 机动目标如启用，只能沿防护区内道路模拟移动。
- [ ] 起降点 / 观察点可设置。
- [ ] 覆盖规划可生成覆盖/采集建议。
- [ ] 覆盖路线可显示在地图上。
- [ ] 盲区 / 未覆盖区可显示。
- [ ] 巡查/撤离路径可生成。
- [ ] 诱饵布设可显示并进入组织方裁判视图。
- [ ] 覆盖规划可引用搜索方变化检测结果作为提示。
- [ ] 防护方方案可进入报告。

**W4 Go/No-Go**：搜救路线规划、搜索区更新和 Hide and Seek 防护方覆盖规划均可演示。

## 5. W5：三方联动 + 裁判/完成条件 + 报告

### 5.1 搜救闭环

- [ ] 组织方可创建活动区、搜索区、安全区。
- [ ] 组织方可预置目标点和疑似点，搜索方初始不可见。
- [ ] 搜索方无人机扫描后生成可疑点。
- [ ] 搜索方可复核可疑点并更新优先级。
- [ ] 搜索区可根据复核结果更新。
- [ ] 路径可根据最高优先级可疑点重新规划。
- [ ] 搜索方到达目标点后可标记完成。
- [ ] 未找到时可输出“需扩大搜索区/继续搜索”报告。

### 5.2 Hide and Seek 闭环

- [ ] 组织方可创建活动区、搜索区、防护区、安全区、禁入区。
- [ ] 组织方可设置搜索方资源参数：无人机高度、续航、换电时间、相机参数。
- [ ] 组织方可设置防护方目标点、诱饵和规则。
- [ ] 防护方可提交防护/覆盖方案。
- [ ] 搜索方可执行无人机扫描并接收检测结果。
- [ ] 组织方可裁判确认有效发现。
- [ ] 时间、发现阈值、违规条件可进入结算。
- [ ] 可生成胜负/复盘报告。

### 5.3 通用报告

- [ ] 可导出组织方总览报告。
- [ ] 报告包含地图截图、搜索方发现、防护方方案、路径建议、合规说明。
- [ ] 报告包含真实 vs 模拟披露。
- [ ] 报告明确不包含飞控控制或自动航线能力。

**W5 Go/No-Go**：搜救和 Hide and Seek 主要任务闭环均可完整演示。

## 9. 转入正式工程的条件

满足以下条件后，才进入正式工程 PLAN / BOM / CHECKLIST：

- [ ] MVP 完成至少 3 次外部演示。
- [ ] 至少 2 类目标用户认可三方叙事。
- [ ] 搜救和 Hide and Seek 至少有一个场景被验证有明确客户价值。
- [ ] 明确首个商业主场景：Hide and Seek / Milsim / 安保训练 / 野外搜救 / 巡检之一。
- [ ] 明确正式硬件形态：Backpack / Team / Field 的优先级。
- [ ] 明确是否需要热成像、RTK、LoRa、加固箱。
- [ ] 明确预算和融资节点。
- [ ] 通过 CCB 决定进入 M1 Spec 冻结。
