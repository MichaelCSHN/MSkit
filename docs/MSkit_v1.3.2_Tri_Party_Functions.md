# MSkit v1.3.2 黑 / 红 / 蓝三方架构与功能矩阵

> **定位**：用于活动、游戏、赛事、救援演练、安保巡检等民用/商用场景的角色架构说明。  
> **用途**：明确黑方、红方、蓝方的通用功能、专有功能、权限边界，以及 MVP 路演应优先展示的核心功能。  
> **合规边界**：三方架构只用于民用活动组织、模拟竞赛、搜救训练、安保巡检和户外团队协作；不得解释为军事指挥、武器化、打击排序或作战控制。

---

## 1. 三方定义

| 角色 | 定义 | 典型用户 | 核心目标 | MVP 展示重点 |
|------|------|----------|----------|--------------|
| 黑方 | 活动、游戏、赛事、演练或任务的组织者 / 裁判 / 调度方，统揽全局 | 主办方、裁判组、赛事导演、救援演练指挥员、安保项目经理 | 规则设定、全局态势、PNT 与路径规划、裁判判定、安全边界、报告归档 | PNT、全局地图、路线/区域规划、红蓝状态总览、报告导出 |
| 红方 | 游戏/竞赛中的攻方，或搜救小队、巡检搜索小队 | 竞赛进攻队、搜救队、巡检队、生态观察队 | 检测、搜索、识别、标注、证据采集、搜索进度推进 | GOD / COD、搜索轨迹、发现点标注、目标候选列表 |
| 蓝方 | 游戏/竞赛中的守方，或安保小队、防护/值守小队 | 竞赛防守队、安保队、场地值守队、设施巡护队 | 覆盖设计、区域防护、变化监测、防守/值守部署、风险响应 | CCD 方案设计、保护区/禁入区、覆盖缺口、值守路径建议 |

## 2. 通用功能

以下功能三方均可使用，但权限和视图不同。黑方通常拥有全局视图；红方和蓝方只看到授权区域、己方任务和必要的共享态势。

| 功能域 | 通用功能 | 黑方权限 | 红方权限 | 蓝方权限 |
|--------|----------|----------|----------|----------|
| 账户与角色 | 登录、角色切换、队伍成员管理、设备绑定 | 全局管理 | 己方成员/设备 | 己方成员/设备 |
| 地图与 PNT | 离线地图、当前位置、轨迹、时间线、坐标标注 | 全局 PNT 与全部轨迹 | 己方轨迹 + 授权共享点 | 己方轨迹 + 防护区域态势 |
| 区域管理 | 活动区域、禁入区、安全区、任务区 | 创建/修改/冻结 | 查看与执行 | 查看、建议、执行 |
| 数据接入 | 无人机视频、图片、飞行日志、手工标注、传感器数据 | 全局导入与归档 | 己方数据导入 | 己方数据导入 |
| 任务协作 | 任务领取、状态上报、事件标注、评论 | 创建、分派、裁决 | 执行和上报 | 执行和上报 |
| 路径规划 | 起点/目标/禁入区路径建议、覆盖路线 | 全局规划与审批 | 搜索路径建议 | 值守/防护路径建议 |
| 事件与告警 | 发现点、变化点、越界、异常、风险提示 | 全局接收和裁决 | 上报发现/异常 | 上报防护/越界/变化 |
| 报告 | 活动报告、队伍报告、事件证据包 | 全局报告与导出 | 己方任务报告 | 己方防护报告 |
| 审计与回放 | 操作日志、轨迹回放、事件回放 | 全量回放 | 己方回放 | 己方回放 |
| 合规与隐私 | 数据脱敏、导出确认、素材许可证、知情同意 | 全局责任 | 执行约束 | 执行约束 |

## 3. 黑方专有功能

黑方是“组织者/裁判/调度方”，核心不是检测或防护，而是全局态势、规则、裁决与安全边界。

| 功能 | 说明 | MVP 是否展示 |
|------|------|--------------|
| 活动 / 赛事配置 | 创建活动、设置时间、区域、规则、队伍、设备、评分方式 | 是，简化展示 |
| 三方角色编组 | 创建黑/红/蓝角色，绑定人员、设备、无人机素材 | 是 |
| 全局 PNT 态势 | 显示红/蓝轨迹、无人机轨迹、关键事件位置 | 是，核心展示 |
| 区域与边界冻结 | 设置活动区、禁入区、安全区、蓝方保护区、红方搜索区 | 是 |
| 路线与任务审批 | 对红方搜索路线、蓝方覆盖方案进行查看和确认 | 是，简化展示 |
| 裁判 / 判定面板 | 对发现点、变化点、越界事件、任务完成状态做裁决 | MVP 可模拟 |
| 计分与结果 | 游戏/赛事积分、用时、覆盖率、发现数量、误报/漏报摘要 | MVP 可模拟 |
| 全局时间线 | 按时间查看事件、轨迹、检测、变化、报告 | 是 |
| 事件回放 | 回放红蓝双方轨迹和事件 | MVP 可简化 |
| 安全管控 | 越界提醒、飞行区域提醒、隐私/禁拍区域提示 | 是 |
| 报告归档 | 输出活动总报告、队伍报告、裁判摘要 | 是 |

## 4. 红方专有功能

红方代表“搜索/检测/识别”能力。在游戏/竞赛中可代表攻方；在救援和巡检中可代表搜救小队或搜索小队。

| 功能 | 说明 | MVP 是否展示 |
|------|------|--------------|
| GOD 地面对象检测 | 从无人机视频/图片中检测人员、车辆、动物、设施、异常热源等民用目标 | 是，核心展示 |
| COD 变化/状态检测 | 对比基线与当前影像，识别新增、消失、移动、异常区域 | 是，核心展示 |
| 搜索轨迹 | 显示搜索路线、覆盖进度、未搜索区域 | 是 |
| 发现点标注 | 对检测结果进行确认、驳回、补充说明、证据截图 | 是 |
| 候选目标列表 | 按时间、区域、置信度展示候选发现 | 是 |
| 搜索路径建议 | 根据黑方区域和目标点生成搜索路线建议 | 是，简化展示 |
| 证据包生成 | 将发现点、截图、时间戳、坐标、说明打包 | 是 |
| 多媒体复核 | 视频帧、图片、热成像素材复核 | MVP 可用样例素材 |
| 搜索任务状态 | 未开始、进行中、已覆盖、已复核、已提交 | 是 |
| 搜救小队模式 | 将“目标发现”替换为“人员/动物/风险点发现” | 是，路演推荐使用 |

---

## 5. 蓝方专有功能

蓝方代表“防护/守方/安保/值守”能力。核心不是攻击，而是覆盖设计、边界维护、变化监测和防护响应。

| 功能 | 说明 | MVP 是否展示 |
|------|------|--------------|
| CCD 覆盖与采集方案设计 | 设计观测点、巡查路线、覆盖区域、采集频次和预计耗时 | 是，核心展示 |
| 防护区域设置 | 设置保护区、禁入区、缓冲区、观察区 | 是 |
| 覆盖缺口分析 | 显示未覆盖区域、盲区、重复覆盖区 | 是 |
| 值守 / 巡查路径建议 | 为蓝方小队生成巡查路径、观察点顺序 | 是 |
| 变化监测 | 对重点区域做 COD 变化提醒 | 是，复用 COD 但视角归蓝方 |
| 越界 / 进入提醒 | 对授权边界、禁入区、保护区变化进行提醒 | MVP 可模拟 |
| 防护资源布局 | 人员、无人机观察点、摄像机点位、巡查节点布局建议 | MVP 简化展示 |
| 风险响应建议 | 当出现变化点或越界事件时，生成最近路径和处置建议 | 是，简化展示 |
| 守方任务状态 | 防护中、巡查中、发现异常、已处置、已提交 | 是 |
| 安保小队模式 | 将蓝方能力表达为活动安保、设施巡护、边界维护 | 是，路演推荐使用 |

| 数据 / 能力 | 黑方 | 红方 | 蓝方 |
|-------------|------|------|------|
| 全局地图 | 可见 | 部分可见 | 部分可见 |
| 全部红蓝轨迹 | 可见 | 仅己方或授权共享 | 仅己方或授权共享 |
| 无人机原始素材 | 可归档 | 己方上传/授权素材 | 己方上传/授权素材 |
| GOD/COD 结果 | 可见并裁决 | 可创建/复核 | 可查看授权结果，重点查看防护区变化 |
| CCD 方案 | 可审批/归档 | 可查看授权结果 | 可创建/调整 |
| 路径规划 | 全局规划/审批 | 搜索路径 | 防护/巡查路径 |

| 规则配置 | 可修改 | 只读 | 只读 |
| 评分/判定 | 可裁决 | 查看结果 | 查看结果 |

## 7. MVP 展示取舍

MVP 不展示全部三方功能，而是各取一个代表性核心能力，形成清晰路演叙事。

| 方别 | MVP 核心展示 | 代表价值 | 不做事项 |
|------|--------------|----------|----------|
| 黑方 | PNT 与全局规划：活动区域、红蓝轨迹、无人机轨迹、路径建议、报告 | 证明系统能统揽全局、组织活动、裁判/调度/归档 | 不做复杂计分系统，不做真实 RTK 精度承诺 |
| 红方 | GOD/COD：检测、搜索、识别、变化发现、候选点复核 | 证明系统能从无人机数据中产生“发现” | 不做武器识别、攻击意图分类、打击排序 |
| 蓝方 | CCD 方案设计：防护区、禁入区、覆盖路线、盲区分析、响应路径 | 证明系统能做防护和覆盖设计 | 不做自动化防御、不做对抗性控制 |

MVP 推荐演示闭环：

1. 黑方创建活动区域、红蓝队伍和安全边界。
2. 红方导入无人机视频/日志，运行 GOD/COD，生成发现点和变化点。
3. 蓝方基于保护区和变化点生成 CCD 覆盖方案与巡查路径。
4. 黑方查看 PNT 全局态势，比较红方发现与蓝方覆盖，生成裁判/调度报告。

## 8. 产品路线映射

| MVP 三方能力 | 正式 MSkit 能力 | 后续增强 |
|--------------|----------------|----------|
| 黑方 PNT / 全局规划 | MAP-NAV、TASK/SIM、REPORT、审计回放 | RTK、LoRa 多节点、权限、裁判计分、事件回放 |
| 红方 GOD/COD | SENSE、AI Pipeline、媒体证据包 | TensorRT、热成像、多类别数据集、mAP/FPS 验收 |
| 蓝方 CCD | 任务规划、覆盖设计、区域防护、风险响应 | 多节点巡查、覆盖优化、告警策略、设施/边界长期监测 |

## 9. 合规措辞建议

推荐对外使用：

- “活动组织者 / 裁判视图 / 全局调度视图”描述黑方。
- “搜索小队 / 搜救小队 / 巡检小队 / 竞赛攻方”描述红方。
- “安保小队 / 值守小队 / 防护方 / 竞赛守方”描述蓝方。
- “覆盖设计、采集建议、路径建议、态势展示”描述规划能力。

避免对外使用：

- 作战、打击、攻击目标、武器识别、敌我识别、火力、歼灭、压制等军事化表述。
- 自动飞控、自动航线下发、自主追踪等可能引发无人机合规争议的表述。
