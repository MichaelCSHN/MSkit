# MSkit v1.3.2 组织 / 搜索 / 防护三方架构与功能矩阵

> **定位**：用于 Hide and Seek 实景对抗赛、大型 Milsim 实景游戏、安保训练、野外搜救、灾害演练、户外赛事和巡检场景的角色架构说明。  
> **用途**：明确组织方、搜索方、防护方的通用功能、专有功能、权限边界，以及 MVP 路演应优先展示的核心功能。  
> **合规边界**：三方架构只用于民用活动组织、模拟竞赛、搜救训练、安保巡检和户外团队协作；不得解释为军事指挥、武器化、打击排序或作战控制。
>
> **权威性**：本文为**组织 / 搜索 / 防护三方角色定义与功能矩阵的权威来源**；README、Positioning、MVP 与开发计划等文档引用本文，不重复完整定义。命名采用中性角色模型（不用红/蓝/黑或攻/守），能力术语统一为目标检测 / 变化检测 / 覆盖规划 / 定位态势。

---

## 1. 三方定义

| 角色 | 定义 | 典型用户 | 核心目标 | MVP 展示重点 |
|------|------|----------|----------|--------------|
| 组织方 | 活动、游戏、赛事、演练或任务的组织者 / 裁判 / 调度方，统揽全局 | 主办方、裁判组、赛事导演、教官、救援协调员、安保项目经理 | 规则设定、全局态势、定位态势与路径规划、裁判判定、安全边界、报告归档 | 定位态势、全局地图、路线/区域规划、搜索/防护状态总览、报告导出 |
| 搜索方 | 游戏/竞赛中的寻找方，或搜救小队、巡检搜索小队 | 竞赛进攻队、搜救队、巡检队、生态观察队 | 检测、搜索、识别、标注、证据采集、搜索进度推进 | 目标检测 / 变化检测、搜索轨迹、发现点标注、目标候选列表 |
| 防护方 | 游戏/竞赛中的防守方，或安保小队、防护/值守小队 | 竞赛防守队、安保队、场地值守队、设施巡护队 | 覆盖设计、区域防护、变化监测、防守/值守部署、风险响应 | 覆盖规划方案设计、保护区/禁入区、覆盖缺口、值守路径建议 |

## 2. 三方与应用场景映射

| 场景 | 三方结构 | 组织方任务 | 搜索方任务 | 防护方任务 |
|------|----------|----------|----------|----------|
| Hide and Seek 实景对抗赛 | 组织 / 搜索 / 防护 | 创建活动、设定边界和规则、裁判计时、输出结果 | 搜索隐藏目标/线索，运行 目标检测/变化检测，提交发现证据 | 设计隐藏/防护区域，规划覆盖路线，分析盲区 |
| 大型 Milsim 实景游戏 | 组织 / 搜索 / 防护 | 主办、安全裁判、队伍编组、事件裁决、赛后复盘 | 搜索、识别、路线推进、证据采集 | 防护区域、巡查路线、重点区域变化监测 |
| 安保训练 | 组织 / 搜索 / 防护 | 教官/评估员，定义训练区、评分标准和安全边界 | 搜索异常人员/车辆/风险点 | 设计保护区、巡查路径、响应路线和盲区修正 |
| 野外搜救 | 组织 / 搜索 | 搜救协调、搜索区划分、定位态势、报告归档 | 搜救搜索、目标检测/变化检测、发现点证据包 | 可选：营地/安全区值守和防护方边界维护 |
| 灾害应急演练 | 组织 / 搜索 / 防护 | 演练组织、全局调度、物资区与禁入区管理 | 搜索受困点/风险点 | 维护安全区、物资区、通行边界 |
| 越野定向 / 户外挑战赛 | 组织 / 搜索 / 防护 | 赛事组织、检查点、计时和成绩 | 寻找检查点和路线证据 | 安全区、补给点、风险区域巡查 |
| 大型活动安保 | 组织 / 防护，可选搜索方 | 主办方、安保调度、事件归档 | 可选巡检搜索队 | 值守、巡查、禁入区和保护区覆盖 |
| 工业 / 农林巡检 | 组织 / 搜索 / 防护 | 运营管理和报告归档 | 巡检发现异常 | 维护边界、重点设施和值守路线 |
| 生态观察与动物追踪 | 组织 / 搜索 | 观察任务和区域管理 | 动物/足迹/热源证据采集 | 通常不设防护方；如有保护区可设置简化防护方 |

## 3. 通用功能

以下功能三方均可使用，但权限和视图不同。组织方通常拥有全局视图；搜索方和防护方只看到授权区域、己方任务和必要的共享态势。

| 功能域 | 通用功能 | 组织方权限 | 搜索方权限 | 防护方权限 |
|--------|----------|----------|----------|----------|
| 账户与角色 | 登录、角色切换、队伍成员管理、设备绑定 | 全局管理 | 己方成员/设备 | 己方成员/设备 |
| 地图与 定位态势 | 离线地图、当前位置、轨迹、时间线、坐标标注 | 全局定位态势与全部轨迹 | 己方轨迹 + 授权共享点 | 己方轨迹 + 防护区域态势 |
| 区域管理 | 活动区域、禁入区、安全区、任务区 | 创建/修改/冻结 | 查看与执行 | 查看、建议、执行 |
| 数据接入 | 无人机视频、图片、飞行日志、手工标注、传感器数据 | 全局导入与归档 | 己方数据导入 | 己方数据导入 |
| 任务协作 | 任务领取、状态上报、事件标注、评论 | 创建、分派、裁决 | 执行和上报 | 执行和上报 |
| 路径规划 | 起点/目标/禁入区路径建议、覆盖路线 | 全局规划与审批 | 搜索路径建议 | 值守/防护路径建议 |
| 事件与告警 | 发现点、变化点、越界、异常、风险提示 | 全局接收和裁决 | 上报发现/异常 | 上报防护/越界/变化 |
| 报告 | 活动报告、队伍报告、事件证据包 | 全局报告与导出 | 己方任务报告 | 己方防护报告 |
| 审计与回放 | 操作日志、轨迹回放、事件回放 | 全量回放 | 己方回放 | 己方回放 |
| 合规与隐私 | 数据脱敏、导出确认、素材许可证、知情同意 | 全局责任 | 执行约束 | 执行约束 |

## 5. 搜索方专有功能

| 功能 | 说明 | MVP 是否展示 |
|------|------|--------------|
| 目标检测（地面民用目标检测） | 从无人机视频/图片中检测人员、车辆、动物、设施、异常热源等民用目标 | 是，核心展示 |
| 变化检测（变化/状态检测） | 对比基线与当前影像，识别新增、消失、移动、异常区域 | 是，核心展示 |
| 搜索轨迹 | 显示搜索路线、覆盖进度、未搜索区域 | 是 |
| 发现点标注 | 对检测结果进行确认、驳回、补充说明、证据截图 | 是 |
| 候选目标列表 | 按时间、区域、置信度展示候选发现 | 是 |
| 搜索路径建议 | 根据组织方区域和目标点生成搜索路线建议 | 是，简化展示 |
| 证据包生成 | 将发现点、截图、时间戳、坐标、说明打包 | 是 |
| 多媒体复核 | 视频帧、图片、热成像素材复核 | MVP 可用样例素材 |

| 搜救小队模式 | 将“目标发现”替换为“人员/动物/风险点发现” | 是，路演推荐使用 |

| 防护区域设置 | 设置保护区、禁入区、缓冲区、观察区 | 是 |

## 7. 三方数据与权限边界

| 数据 / 能力 | 组织方 | 搜索方 | 防护方 |
|-------------|------|------|------|
| 全局地图 | 可见 | 部分可见 | 部分可见 |
| 全部搜索/防护轨迹 | 可见 | 仅己方或授权共享 | 仅己方或授权共享 |
| 无人机原始素材 | 可归档 | 己方上传/授权素材 | 己方上传/授权素材 |
| 目标检测/变化检测 结果 | 可见并裁决 | 可创建/复核 | 可查看授权结果，重点查看防护区变化 |
| 覆盖规划 方案 | 可审批/归档 | 可查看授权结果 | 可创建/调整 |
| 路径规划 | 全局规划/审批 | 搜索路径 | 防护/巡查路径 |
| 报告导出 | 全局报告 | 己方报告 | 己方报告 |
| 规则配置 | 可修改 | 只读 | 只读 |
| 评分/判定 | 可裁决 | 查看结果 | 查看结果 |

## 8. MVP 展示取舍

| 方别 | MVP 核心展示 | 代表价值 | 不做事项 |
|------|--------------|----------|----------|
| 组织方 | 定位态势与全局规划：活动区域、搜索/防护轨迹、无人机轨迹、路径建议、报告 | 证明系统能统揽全局、组织活动、裁判/调度/归档 | 不做复杂计分系统，不做真实 RTK 精度承诺 |
| 搜索方 | 目标检测/变化检测：检测、搜索、识别、变化发现、候选点复核 | 证明系统能从无人机数据中产生“发现” | 不做武器识别、攻击意图分类、打击排序 |
| 防护方 | 覆盖规划方案设计：防护区、禁入区、覆盖路线、盲区分析、响应路径 | 证明系统能做防护和覆盖设计 | 不做自动化防御、不做对抗性控制 |

MVP 主脚本建议采用 **Hide and Seek 实景对抗赛**；备选脚本采用 **野外搜救**。

## 9. 产品路线映射

## 10. 合规措辞建议

推荐对外使用：

- “活动组织者 / 裁判视图 / 全局调度视图”描述组织方。
- “搜索小队 / 搜救小队 / 巡检小队 / 竞赛搜索方”描述搜索方。
- “安保小队 / 值守小队 / 防护方 / 竞赛防护方”描述防护方。
- “覆盖设计、采集建议、路径建议、态势展示”描述规划能力。

避免对外使用：

- 作战、打击、攻击目标、武器识别、敌我识别、火力、歼灭、压制等军事化表述。
- 自动飞控、自动追踪、自动航线下发等可能引发无人机合规争议的表述。
