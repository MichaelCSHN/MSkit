# MSkit v1.3.2 MVP 演示开发计划（立项 / 融资路演）

> **定位**：用于立项演示、融资路演、早期客户访谈的轻量 MVP，不替代 v1.4 工程样机。  
> **目标**：用最小硬件和最短周期展示 MSkit 的三方核心闭环：黑方全局 PNT/规划 → 红方 GOD/COD 检测搜索 → 蓝方 CCD 覆盖防护 → 黑方裁判/调度报告。  
> **硬件边界**：优先采用“笔记本 + 无人机遥控器 + 无人机”；可选“笔记本 + 无人机/离线素材导入”。MVP 阶段不做无人机飞控控制、不下发飞行指令、不做自主飞行。

---

## 1. MVP 目标

MVP 的目的不是证明全部硬件规格，而是证明以下商业与产品假设：

1. 用户能理解 MSkit 是一个支持黑/红/蓝三方协作的便携边缘智能信息中心。
2. 黑方可以统揽全局：活动区域、红蓝队伍、PNT 轨迹、路径规划、裁判/调度报告。
3. 红方可以完成检测、搜索、识别：通过 GOD/COD 从无人机视频、图片和日志中产生发现点、变化点和搜索证据。
4. 蓝方可以完成 CCD 方案设计：围绕保护区、禁入区和重点区域生成覆盖/采集方案、盲区提示和巡查路径建议。
5. 不依赖加固箱、Jetson、LoRa Mesh、RTK 硬件，也能在笔记本上展示核心软件体验。
6. 演示全程保持民用/商用合规边界，不展示武器化、作战控制或无人机飞控能力。

## 2. 黑 / 红 / 蓝三方定义

| 角色 | 定义 | 核心目标 | MVP 代表功能 |
|------|------|----------|--------------|
| 黑方 | 活动、游戏、赛事、演练或任务的组织者 / 裁判 / 调度方，统揽全局 | 规则、区域、队伍、PNT、路径规划、裁判判定、报告归档 | PNT 与全局规划 |
| 红方 | 游戏/竞赛中的攻方，或搜救小队、巡检搜索小队 | 检测、搜索、识别、发现点标注、证据采集 | GOD / COD |
| 蓝方 | 游戏/竞赛中的守方，或安保小队、防护/值守小队 | CCD、区域防护、覆盖设计、变化监测、巡查路径 | CCD 方案设计 |

MVP 演示取舍：

| 方别 | 必须展示 | 目的 |
|------|----------|------|
| 黑方 | 活动区域、红蓝队伍、无人机/队伍轨迹、PNT、路径规划、报告 | 证明系统能统揽全局和支撑裁判/调度 |
| 红方 | GOD/COD、搜索轨迹、发现点、变化点、候选目标复核 | 证明系统能从无人机数据中产生可解释发现 |
| 蓝方 | CCD、防护区、禁入区、覆盖路线、盲区分析、巡查路径 | 证明系统能做防护与覆盖方案设计 |

## 3. MVP 内部术语定义

> 术语用于演示阶段，v1.4 冻结前可按产品命名再调整。

| 缩写 | 建议定义 | 演示含义 | 边界 |
|------|----------|----------|------|
| GOD | Ground Object Detection / 地面对象检测 | 红方从无人机视频/图片中检测人员、车辆、动物、设施、异常热源等民用目标 | 不做武器、军用目标或攻击意图分类 |
| COD | Change / Condition Object Detection / 变化与状态检测 | 红方或蓝方对比基线与当前图像，识别新增/消失/移动对象、区域状态变化、异常聚集等 | 不做军事打击评估或目标优先级打击排序 |
| CCD | Coverage & Collection Design / 覆盖与采集方案设计 | 蓝方根据保护区、禁入区、起降点、航时估算生成观测点、覆盖路线和采集建议 | 只输出规划建议，不自动控制无人机 |
| PNT | Positioning, Navigation and Timing / 定位、导航与时间 | 黑方用于统一显示轨迹、时间线、发现点、变化点和路径建议 | MVP 可用飞行日志或模拟轨迹，不承诺 RTK 实测精度 |

## 4. MVP 硬件方案

### 4.1 推荐方案 A：笔记本 + 遥控器 + 无人机

| 组件 | 作用 | 要求 |
|------|------|------|
| 笔记本 | 运行 MSkit MVP Web App、AI 推理、地图、路径规划、报告导出 | NVIDIA GPU 笔记本优先；最低可用 CPU 推理 + 离线素材演示 |
| 无人机遥控器 / 原厂 App | 合规操控无人机、显示实时画面、记录飞行日志 | 飞手使用原厂遥控器操作，MSkit 不发送控制指令 |
| 无人机 | 采集视频、图片、GNSS 日志 | 支持导出媒体和飞行日志；热成像可选 |
| 视频接入 | HDMI 采集卡、USB 采集、屏幕共享、RTSP 或离线文件导入 | 优先选择最稳定、最少开发的方式 |
| 地图数据 | MBTiles / OSM 离线瓦片 | 支持演示区域离线地图 |

**优点**：合规边界清晰，演示稳定，可现场飞行或播放预录素材。  
**限制**：MSkit 只做数据接入、态势展示和方案建议，不做飞控。

### 4.2 备选方案 B：笔记本 + 无人机 / 离线素材导入

| 组件 | 作用 | 要求 |
|------|------|------|
| 笔记本 | 运行完整 MVP 软件 | 可离线演示 |
| 无人机或样例素材 | 提供视频、图片、飞行日志 | 可使用预录视频、照片、SRT/GPX/CSV 日志 |
| 数据导入工具 | 批量导入媒体和日志 | 保证路演不依赖现场飞行条件 |

**优点**：最稳定，适合融资路演、室内演示、政策或天气不确定场景。  
**限制**：实时感弱，但可通过录屏或回放方式展示完整闭环。

## 5. MVP 功能范围

### 5.1 三方通用功能（MVP Core）

| 模块 | 功能 | 演示结果 |
|------|------|----------|
| 角色视图 | 黑方 / 红方 / 蓝方视图切换 | 观众能理解三方协作模型 |
| 数据接入 | 导入无人机视频、图片、飞行日志；可选实时视频采集 | 媒体与轨迹在同一项目中可浏览 |
| 地图态势 | 离线地图、飞行轨迹、红蓝任务区、检测结果点位、区域边界 | 观众能看到“谁在什么区域发现了什么” |
| PNT | 轨迹、时间线、事件点、路径建议 | 黑方可统揽全局，红蓝可查看授权态势 |
| 报告导出 | 导出 HTML/PDF/Markdown 演示报告 | 包含地图截图、红方发现、蓝方方案、黑方总结 |

### 5.2 红方核心功能：GOD / COD

| 模块 | 功能 | 演示结果 |
|------|------|----------|
| GOD | 对视频/图片进行地面对象检测 | 检测框、类别、置信度、时间戳、地图点位 |
| COD | 对比基线与当前影像，识别变化对象或区域状态变化 | 新增/消失/移动/异常区域高亮 |
| 搜索轨迹 | 展示红方搜索路线和覆盖进度 | 地图上可见红方搜索路径 |
| 发现点复核 | 对候选点确认/驳回/备注 | 生成可解释的发现证据 |
| 红方报告 | 导出发现点、变化点、截图、坐标和时间戳 | 证明搜索/识别闭环 |

### 5.3 蓝方核心功能：CCD 方案设计

| 模块 | 功能 | 演示结果 |
|------|------|----------|
| CCD | 绘制保护区、禁入区、观察区、起降点，生成覆盖与采集建议 | 覆盖率、建议观测点、建议路径、预计耗时 |
| 覆盖缺口 | 显示未覆盖区域、盲区、重复覆盖区 | 蓝方能解释防护方案质量 |
| 巡查路径 | 生成值守/巡查路径建议 | 地图上的蓝方巡查路线 |
| 变化监测 | 在保护区内复用 COD，提示变化点 | 蓝方视角下的异常区域高亮 |
| 蓝方报告 | 导出 CCD 方案、覆盖图、盲区和响应建议 | 证明防护/守方闭环 |

### 5.4 黑方核心功能：PNT 与全局规划

| 模块 | 功能 | 演示结果 |
|------|------|----------|
| 活动配置 | 创建活动区域、红蓝队伍、任务边界 | 形成三方演示上下文 |
| 全局 PNT | 显示红方、蓝方、无人机轨迹与事件时间线 | 黑方掌握全局态势 |
| 路径规划 | 对红方搜索路线和蓝方巡查路线进行查看/调整/确认 | 形成“黑方统筹、红蓝执行”的闭环 |
| 裁判 / 调度视图 | 查看红方发现、蓝方覆盖、越界/变化事件 | MVP 可模拟裁判判定 |
| 总报告 | 生成黑方总览报告 | 适合路演展示商业价值 |

### 5.5 可以模拟（Demo Stub）

| 模块 | 模拟方式 | 目的 |
|------|----------|------|
| 实时视频 | 使用预录视频循环播放 | 避免路演现场飞行风险 |
| 热成像 | 使用样例热成像图片/视频 | 展示 Thermal 版本价值 |
| RTK 精度 | 使用日志中的 GNSS 或手动标注点 | 展示 PNT 体验，不承诺实测 RTK |
| 多节点 Mesh | 用虚拟节点模拟红蓝队伍 | 展示未来 Team/Field 版本场景 |
| 加固硬件 | 用渲染图或产品图说明 | MVP 不做加固箱实物 |
| 裁判计分 | 使用固定规则或手工判定 | 展示黑方统筹能力 |

### 5.6 明确不做（MVP Exclusions）

- 不做无人机飞控控制。
- 不做自动起飞、自动降落、自动航线下发。
- 不做 LoRa Mesh 实物组网。
- 不做 Jetson/加固箱硬件样机。
- 不做 IP65/MIL-STD-810H/EMC 实测。
- 不做真实 RTK <2cm 精度承诺。
- 不做闭源 YOLO 商业授权冻结；MVP 可使用内部评估模型或替代开源模型，但路演对外展示前需确认许可证边界。
- 不做军事化目标分类、武器识别、打击排序或作战控制。

## 6. MVP 系统架构

```text
[Drone / RC / Media Files]
        |
        |  video / image / logs
        v
[Ingest Service]
        |
        +--> [Media Store]
        +--> [Telemetry Parser / PNT Timeline]
        |
        v
[AI Pipeline: Red GOD / COD]
        |
        v
[Geo Projection + Event Store]
        |
        +--> [Blue CCD Planner]
        +--> [Black PNT / Route Planner]
        |
        v
[Tri-Party Web Dashboard: Black / Red / Blue Views]
        |
        v
[Demo Report Export]
```

### 技术建议

| 层级 | MVP 选型 |
|------|----------|
| 前端 | React / Vite / MapLibre GL，三方视图切换 |
| 后端 | Python FastAPI |
| 数据 | SQLite + 本地文件目录 |
| AI | YOLO 系列或可替代检测模型；MVP 阶段优先保证演示稳定性 |
| 地图 | MBTiles / OSM 离线瓦片 |
| 路径规划 | A* / Dijkstra / 简化覆盖路径算法 |
| 报告 | HTML + Markdown + 可选 PDF 导出 |
| 部署 | 单机 Docker Compose 或 Python venv + Node build |

## 7. 演示脚本

### 7.1 3 分钟融资路演版

1. 打开黑方视图，创建活动区域、红方搜索区、蓝方保护区和安全边界。
2. 导入无人机视频和飞行日志，黑方 PNT 视图显示轨迹和时间线。
3. 切换红方视图，运行 GOD/COD，地图上出现发现点、变化点和候选列表。
4. 切换蓝方视图，基于保护区生成 CCD 覆盖方案、盲区提示和巡查路径。
5. 回到黑方视图，查看红方发现与蓝方覆盖的叠加态势，生成总报告。

### 7.2 8–10 分钟技术演示版

1. 说明黑/红/蓝三方定义和民用/商用边界。
2. 选择硬件方案 A 或 B。
3. 展示黑方：活动配置、PNT、无人机轨迹、红蓝区域。
4. 展示红方：GOD 检测结果如何映射到地图，COD 如何识别变化。
5. 展示蓝方：CCD 方案设计、保护区、禁入区、覆盖率、盲区。
6. 展示定位-路径规划：红方搜索路径、蓝方巡查路径、黑方调度确认。
7. 展示报告导出和审计日志。
8. 说明 MVP 与正式 MSkit 的升级路径：Jetson、RTK、热成像、LoRa、加固箱、认证。

## 9. MVP 验收标准

| 指标 | MVP 验收线 |
|------|------------|
| 演示稳定性 | 连续演示 10 分钟无崩溃 |
| 三方视图 | 黑/红/蓝三方视图或过滤器可切换 |
| 数据接入 | 可导入至少 1 段视频、20 张图片或 1 份飞行日志 |
| 地图显示 | 离线地图加载 <5 秒，轨迹和检测点可显示 |
| 黑方 PNT | 可显示活动区、红蓝区域、无人机轨迹、事件时间线、路径建议 |
| 红方 GOD/COD | 至少 1–3 类民用目标检测可稳定展示；至少支持两组图像/视频片段变化高亮 |
| 蓝方 CCD | 可绘制保护区、禁入区、起降点，并生成覆盖/采集建议和盲区提示 |
| 路径规划 | 可生成绕开禁入区的红方搜索路径和蓝方巡查路径建议 |
| 报告导出 | 可导出包含黑方总览、红方发现、蓝方方案、路径建议的报告 |
| 合规边界 | UI 和脚本不出现军用采购、武器化、作战控制、打击排序等表述 |

## 12. MVP 风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| 三方术语被误解为军事用途 | 合规和融资风险 | 使用民用场景脚本：赛事裁判、救援演练、生态观察、活动安保 |
| 现场飞行受天气/法规/场地影响 | 路演失败 | 必须准备离线素材和录屏备份 |
| 实时视频链路不稳定 | 演示中断 | 默认使用预录视频，实时接入作为加分项 |
| GOD/COD 模型效果不稳定 | 观感差 | 固化演示素材，限定类别，使用置信度阈值和人工复核 |
| CCD 过于抽象 | 观众不理解蓝方价值 | 用保护区、禁入区、盲区和巡查路线可视化解释 |
| 无人机控制边界不清 | 合规风险 | UI 不提供飞控按钮，不下发航线，只输出“建议路径/采集方案” |
| 许可证不清 | 对外展示风险 | 路演前确认模型、地图、样例数据、字体、图标许可证 |

## 13. 立项 / 融资展示重点

MVP 路演应强调：

1. **三方架构清晰**：黑方统揽全局，红方搜索检测，蓝方覆盖防护。
2. **低硬件门槛**：用笔记本和现成无人机即可展示核心价值。
3. **明确升级路径**：从笔记本 MVP 迁移到 Jetson + 加固箱 + RTK + LoRa。
4. **可商业化场景**：活动赛事、安保巡检、救援演练、生态观察、户外团队协调。
