# MSkit v1.3.2 MVP 演示开发计划（立项 / 融资路演）

> **定位**：用于立项演示、融资路演、早期客户访谈的轻量 MVP，不替代 v1.4 工程样机。  
> **目标**：用最小硬件和最短周期展示 MSkit 的核心闭环：无人机/视频数据接入 → GOD/COD 检测 → CCD 方案设计 → 定位-路径规划 → 地图态势展示 → 报告导出。  
> **硬件边界**：优先采用“笔记本 + 无人机遥控器 + 无人机”；可选“笔记本 + 无人机/离线素材导入”。MVP 阶段不做无人机飞控控制、不下发飞行指令、不做自主飞行。

---

## 1. MVP 目标

MVP 的目的不是证明全部硬件规格，而是证明以下商业与产品假设：

1. 用户能理解“便携边缘智能信息中心”的核心价值。
2. 系统能把无人机视频/照片/日志转化为地图态势、目标检测、变化识别和路径建议。
3. GOD/COD + CCD + 定位-路径规划可以形成可演示的决策闭环。
4. 不依赖加固箱、Jetson、LoRa Mesh、RTK 硬件，也能在笔记本上展示核心软件体验。
5. 演示全程保持民用/商用合规边界，不展示武器化、作战控制或无人机飞控能力。

---

## 2. MVP 内部术语定义

> 术语用于演示阶段，v1.4 冻结前可按产品命名再调整。

| 缩写 | 建议定义 | 演示含义 | 边界 |
|------|----------|----------|------|
| GOD | Ground Object Detection / 地面对象检测 | 从无人机视频/图片中检测人员、车辆、动物、设施、异常热源等民用目标 | 不做武器、军用目标或攻击意图分类 |
| COD | Change / Condition Object Detection / 变化与状态检测 | 对比基线与当前图像，识别新增/消失/移动对象、区域状态变化、异常聚集等 | 不做军事打击评估或目标优先级打击排序 |
| CCD | Coverage & Collection Design / 覆盖与采集方案设计 | 根据区域、目标、禁飞/安全边界、起降点、航时估算生成观测点、覆盖路线和采集建议 | 只输出规划建议，不自动控制无人机 |

## 3. MVP 硬件方案

### 3.1 推荐方案 A：笔记本 + 遥控器 + 无人机

| 组件 | 作用 | 要求 |
|------|------|------|
| 笔记本 | 运行 MSkit MVP Web App、AI 推理、地图、路径规划、报告导出 | NVIDIA GPU 笔记本优先；最低可用 CPU 推理 + 离线素材演示 |
| 无人机遥控器 / 原厂 App | 合规操控无人机、显示实时画面、记录飞行日志 | 飞手使用原厂遥控器操作，MSkit 不发送控制指令 |
| 无人机 | 采集视频、图片、GNSS 日志 | 支持导出媒体和飞行日志；热成像可选 |
| 视频接入 | HDMI 采集卡、USB 采集、屏幕共享、RTSP 或离线文件导入 | 优先选择最稳定、最少开发的方式 |
| 地图数据 | MBTiles / OSM 离线瓦片 | 支持演示区域离线地图 |

**优点**：合规边界清晰，演示稳定，可现场飞行或播放预录素材。  
**限制**：MSkit 只做数据接入和态势展示，不做飞控。

### 3.2 备选方案 B：笔记本 + 无人机 / 离线素材导入

| 组件 | 作用 | 要求 |
|------|------|------|
| 笔记本 | 运行完整 MVP 软件 | 可离线演示 |
| 无人机或样例素材 | 提供视频、图片、飞行日志 | 可使用预录视频、照片、SRT/GPX/CSV 日志 |
| 数据导入工具 | 批量导入媒体和日志 | 保证路演不依赖现场飞行条件 |

**优点**：最稳定，适合融资路演、室内演示、政策或天气不确定场景。  
**限制**：实时感弱，但可通过录屏或回放方式展示完整闭环。

## 4. MVP 功能范围

### 4.1 必须实现（MVP Core）

| 模块 | 功能 | 演示结果 |
|------|------|----------|
| 数据接入 | 导入无人机视频、图片、飞行日志；可选实时视频采集 | 媒体与轨迹在同一项目中可浏览 |
| 地图态势 | 离线地图、飞行轨迹、检测结果点位、区域边界 | 观众能看到“在哪里发现了什么” |
| GOD | 对视频/图片进行地面对象检测 | 检测框、类别、置信度、时间戳、地图点位 |
| COD | 对比基线与当前影像，识别变化对象或区域状态变化 | 新增/消失/移动/异常区域高亮 |
| CCD | 绘制任务区域、禁入/安全边界、起降点，生成覆盖与采集建议 | 覆盖率、建议观测点、建议路径、预计耗时 |
| 定位-路径规划 | 基于起点、目标点、禁入区、地形/道路/直线距离生成路径建议 | 地图上的路径线和步骤说明 |
| 报告导出 | 导出 HTML/PDF/Markdown 演示报告 | 包含地图截图、检测摘要、路径建议、风险说明 |

### 4.2 可以模拟（Demo Stub）

| 模块 | 模拟方式 | 目的 |
|------|----------|------|
| 实时视频 | 使用预录视频循环播放 | 避免路演现场飞行风险 |
| 热成像 | 使用样例热成像图片/视频 | 展示 Thermal 版本价值 |
| RTK 精度 | 使用日志中的 GNSS 或手动标注点 | 展示定位体验，不承诺实测 RTK |
| 多节点 Mesh | 用虚拟节点模拟 | 展示未来 Team/Field 版本场景 |
| 加固硬件 | 用渲染图或产品图说明 | MVP 不做加固箱实物 |

### 4.3 明确不做（MVP Exclusions）

- 不做无人机飞控控制。
- 不做自动起飞、自动降落、自动航线下发。
- 不做 LoRa Mesh 实物组网。
- 不做 Jetson/加固箱硬件样机。
- 不做 IP65/MIL-STD-810H/EMC 实测。
- 不做真实 RTK <2cm 精度承诺。
- 不做闭源 YOLO 商业授权冻结；MVP 可使用内部评估模型或替代开源模型，但路演对外展示前需确认许可证边界。
- 不做军事化目标分类、武器识别、打击排序或作战控制。

## 5. MVP 系统架构

```text
[Drone / RC / Media Files]
        |
        |  video / image / logs
        v
[Ingest Service]
        |
        +--> [Media Store]
        +--> [Telemetry Parser]
        |
        v
[AI Pipeline: GOD / COD]
        |
        v
[Geo Projection + Event Store]
        |
        +--> [CCD Planner]
        +--> [Route Planner]
        |
        v
[Web Dashboard: Map + Video + Detections + Plan]
        |
        v
[Demo Report Export]
```

### 技术建议

| 层级 | MVP 选型 |
|------|----------|
| 前端 | React / Vite / MapLibre GL |
| 后端 | Python FastAPI |
| 数据 | SQLite + 本地文件目录 |
| AI | YOLO 系列或可替代检测模型；MVP 阶段优先保证演示稳定性 |
| 地图 | MBTiles / OSM 离线瓦片 |
| 路径规划 | A* / Dijkstra / 简化覆盖路径算法 |
| 报告 | HTML + Markdown + 可选 PDF 导出 |
| 部署 | 单机 Docker Compose 或 Python venv + Node build |

## 6. 演示脚本

### 6.1 3 分钟融资路演版

1. 打开 MSkit MVP 项目，加载演示区域离线地图。
2. 导入无人机视频和飞行日志，系统自动显示轨迹。
3. 运行 GOD，地图上出现检测点和视频检测框。
4. 切换 COD，显示与基线影像相比的变化区域。
5. 进入 CCD，绘制任务区域和禁入区，生成建议采集路线与覆盖率。
6. 点击“生成报告”，输出一页摘要：发现对象、变化点、建议路径、风险提示。

### 6.2 8–10 分钟技术演示版

1. 选择硬件方案 A 或 B。
2. 展示数据接入方式：实时视频或离线视频/日志。
3. 展示 GOD 检测结果如何映射到地图。
4. 展示 COD 对比：基线帧 vs 当前帧。
5. 展示 CCD 方案设计：区域、约束、覆盖率、建议观测点。
6. 展示定位-路径规划：从起点到关键点的路径、绕开禁入区域。
7. 展示报告导出和审计日志。
8. 说明 MVP 与正式 MSkit 的升级路径：Jetson、RTK、热成像、LoRa、加固箱、认证。

## 7. 开发计划（6 周）

| 周期 | 目标 | 交付物 | Go/No-Go |
|------|------|--------|----------|
| W1 | 需求冻结与素材准备 | 演示区域、样例视频/图片/日志、UI 线框、术语冻结 | 路演场景和样例素材确认 |
| W2 | 数据接入与地图底座 | 媒体导入、日志解析、轨迹上图、离线地图 | 能展示“视频 + 轨迹 + 地图” |
| W3 | GOD 检测 MVP | 检测模型接入、检测框、置信度、地图点位 | 至少 1 类目标稳定检测并上图 |
| W4 | COD 与 CCD 初版 | 基线/当前对比、变化高亮、区域绘制、覆盖建议 | 能演示变化识别和采集方案设计 |
| W5 | 定位-路径规划与报告 | 起点/目标/禁入区路径规划、报告导出 | 能生成完整演示报告 |
| W6 | 路演打磨 | Demo 数据固化、异常兜底、讲解脚本、录屏备份 | 3 分钟和 10 分钟演示均可稳定运行 |

## 8. MVP 验收标准

| 指标 | MVP 验收线 |
|------|------------|
| 演示稳定性 | 连续演示 10 分钟无崩溃 |
| 数据接入 | 可导入至少 1 段视频、20 张图片或 1 份飞行日志 |
| 地图显示 | 离线地图加载 <5 秒，轨迹和检测点可显示 |
| GOD | 至少 1–3 类民用目标检测可稳定展示 |
| COD | 至少支持两组图像/视频片段变化高亮 |
| CCD | 可绘制区域、禁入区、起降点，并生成覆盖/采集建议 |
| 路径规划 | 可生成绕开禁入区的路径建议 |
| 报告导出 | 可导出包含地图截图、检测摘要和路径建议的报告 |
| 合规边界 | UI 和脚本不出现军用采购、武器化、作战控制、打击排序等表述 |

## 10. 与正式 MSkit 路线的关系

| MVP 能力 | 正式 MSkit 对应能力 | 后续升级 |
|----------|---------------------|----------|
| 笔记本单机推理 | Jetson 边缘推理 | 迁移到 Orin Nano/NX/AGX |
| 离线媒体/日志导入 | 多源传感器实时接入 | 增加热成像、RTK、LoRa、WebSocket |
| GOD/COD 演示模型 | 冻结测试集 + TensorRT engine | 补 mAP、FPS、误报/漏报评估 |
| CCD 采集方案设计 | Team/Field 任务管理与规划 | 加入用户、权限、审计、任务模板 |
| 路径规划建议 | MAP-NAV 模块 | 加入 DEM、道路/地形约束、离线瓦片优化 |
| 演示报告 | REPORT 模块 | 加密导出、签名清单、模板系统 |

## 11. MVP 风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| 现场飞行受天气/法规/场地影响 | 路演失败 | 必须准备离线素材和录屏备份 |
| 实时视频链路不稳定 | 演示中断 | 默认使用预录视频，实时接入作为加分项 |
| GOD/COD 模型效果不稳定 | 观感差 | 固化演示素材，限定类别，使用置信度阈值和人工复核 |
| 术语被误解为军事用途 | 合规和融资风险 | 使用民用场景脚本：巡检、救援演练、生态观察、活动安保 |
| 无人机控制边界不清 | 合规风险 | UI 不提供飞控按钮，不下发航线，只输出“建议路径/采集方案” |
| 许可证不清 | 对外展示风险 | 路演前确认模型、地图、样例数据、字体、图标许可证 |

## 12. 立项 / 融资展示重点

MVP 路演应强调：

1. **低硬件门槛**：用笔记本和现成无人机即可展示核心价值。
2. **明确升级路径**：从笔记本 MVP 迁移到 Jetson + 加固箱 + RTK + LoRa。
3. **可商业化场景**：安保巡检、救援演练、生态观察、户外团队协调。
4. **产品护城河**：GOD/COD + CCD + 定位-路径规划 + 离线地图 + 报告的闭环，而不是单点 AI 检测。
5. **合规边界清晰**：MVP 不做飞控控制，不做武器化或作战用途。
