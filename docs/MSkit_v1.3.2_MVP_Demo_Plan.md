# MSkit v1.3.2 MVP 演示开发计划（立项 / 融资路演）

> **定位**：用于立项演示、融资路演、早期客户访谈的轻量 MVP，不替代 v1.4 工程样机。  
> **目标**：用最小硬件和最短周期展示 MSkit 的三方核心闭环：组织方全局定位态势/规划 → 搜索方 目标检测/变化检测 → 防护方 覆盖规划与防护 → 组织方裁判/调度报告。  
> **主场景**：Hide and Seek 实景对抗赛；**备选场景**：野外搜救。  
> **硬件边界**：优先采用“笔记本 + 无人机遥控器 + 无人机”；可选“笔记本 + 无人机/离线素材导入”。MVP 阶段不做无人机飞控控制、不下发飞行指令、不做自主飞行。

---

## 1. MVP 目标

MVP 的目的不是证明全部硬件规格，而是证明以下商业与产品假设：

1. 用户能理解 MSkit 是一个支持组织/搜索/防护三方协作的实景智能规控平台。
2. Hide and Seek 实景对抗赛、大型 Milsim 实景游戏、安保训练和野外搜救可以共享同一套三方底座。
3. 组织方可以统揽全局：活动区域、搜索/防护队伍、定位态势 轨迹、路径规划、裁判/调度报告。
4. 搜索方可以完成检测、搜索、识别：通过 目标检测/变化检测 从无人机视频、图片和日志中产生发现点、变化点和搜索证据。
5. 防护方可以完成 覆盖规划方案设计：围绕保护区、禁入区和重点区域生成覆盖/采集方案、盲区提示和巡查路径建议。
6. 不依赖加固箱、Jetson、LoRa Mesh、RTK 硬件，也能在笔记本上展示核心软件体验。
7. 演示全程保持民用/商用合规边界，不展示武器化、作战控制或无人机飞控能力。

## 2. MVP 场景选择

### 2.1 主脚本：Hide and Seek 实景对抗赛

| 方别 | 演示角色 | MVP 功能 |
|------|----------|----------|
| 组织方 | 主办方 / 裁判 / 计时与边界管理 | 创建活动区、搜索/防护队伍、安全边界、定位态势时间线、裁判报告 |
| 搜索方 | 搜索方 | 导入无人机视频/日志，运行 目标检测/变化检测，生成发现点、变化点和候选目标列表 |
| 防护方 | 隐藏/防防护方 | 创建隐藏区/保护区，运行 覆盖规划，生成覆盖路线、盲区提示和巡查路径 |

主脚本优势：三方关系清楚，游戏性强，适合融资路演解释产品价值。

### 2.2 备选脚本：野外搜救

| 方别 | 演示角色 | MVP 功能 |
|------|----------|----------|
| 组织方 | 搜救协调员 / 现场指挥 | 创建搜索区、优先区域、禁入区、定位态势时间线和搜救报告 |
| 搜索方 | 搜救小队 | 运行 目标检测/变化检测，标注疑似人员/动物/风险点，提交证据包 |
| 防护方 | 可选：营地/安全区值守 | 如需展示防护方，可设置营地安全区、补给点和禁入边界 |

备选脚本优势：社会价值强，适合非游戏、公益、安全管理类投资人或客户。

| 防护方 | 游戏/竞赛中的防守方，或安保小队、防护/值守小队 | 覆盖规划、区域防护、覆盖设计、变化监测、巡查路径 | 覆盖规划方案设计 |

| 方别 | 必须展示 | 目的 |

| 防护方 | 覆盖规划、防护区、禁入区、覆盖路线、盲区分析、巡查路径 | 证明系统能做防护与覆盖方案设计 |

## 4. MVP 内部术语定义

| 缩写 | 建议定义 | 演示含义 | 边界 |
|------|----------|----------|------|
| 目标检测 | 地面民用目标检测 | 搜索方从无人机视频/图片中检测人员、车辆、动物、设施、异常热源等民用目标 | 不做武器、军用目标或攻击意图分类 |
| 变化检测 | 变化与状态检测 | 搜索方或防护方对比基线与当前图像，识别新增/消失/移动对象、区域状态变化、异常聚集等 | 不做军事打击评估或目标优先级打击排序 |
| 覆盖规划 | 覆盖与采集方案设计 | 防护方根据保护区、禁入区、起降点、航时估算生成观测点、覆盖路线和采集建议 | 只输出规划建议，不自动控制无人机 |
| 定位态势 | 定位、轨迹与时间线 | 组织方用于统一显示轨迹、时间线、发现点、变化点和路径建议 | MVP 可用飞行日志或模拟轨迹，不承诺 RTK 实测精度 |

## 5. MVP 硬件方案

### 5.1 推荐方案 A：笔记本 + 遥控器 + 无人机

| 组件 | 作用 | 要求 |
|------|------|------|
| 笔记本 | 运行 MSkit MVP Web App、AI 推理、地图、路径规划、报告导出 | NVIDIA GPU 笔记本优先；最低可用 CPU 推理 + 离线素材演示 |
| 无人机遥控器 / 原厂 App | 合规操控无人机、显示实时画面、记录飞行日志 | 飞手使用原厂遥控器操作，MSkit 不发送控制指令 |
| 无人机 | 采集视频、图片、GNSS 日志 | 支持导出媒体和飞行日志；热成像可选 |
| 视频接入 | HDMI 采集卡、USB 采集、屏幕共享、RTSP 或离线文件导入 | 优先选择最稳定、最少开发的方式 |
| 地图数据 | MBTiles / OSM 离线瓦片 | 支持演示区域离线地图 |

### 5.2 备选方案 B：笔记本 + 无人机 / 离线素材导入

| 组件 | 作用 | 要求 |
|------|------|------|
| 笔记本 | 运行完整 MVP 软件 | 可离线演示 |
| 无人机或样例素材 | 提供视频、图片、飞行日志 | 可使用预录视频、照片、SRT/GPX/CSV 日志 |
| 数据导入工具 | 批量导入媒体和日志 | 保证路演不依赖现场飞行条件 |

## 6. MVP 功能范围

### 6.1 三方通用功能（MVP Core）

| 模块 | 功能 | 演示结果 |
|------|------|----------|
| 角色视图 | 组织方 / 搜索方 / 防护方视图切换 | 观众能理解三方协作模型 |
| 数据接入 | 导入无人机视频、图片、飞行日志；可选实时视频采集 | 媒体与轨迹在同一项目中可浏览 |
| 地图态势 | 离线地图、飞行轨迹、搜索/防护任务区、检测结果点位、区域边界 | 观众能看到“谁在什么区域发现了什么” |
| 定位态势 | 轨迹、时间线、事件点、路径建议 | 组织方可统揽全局，搜索/防护可查看授权态势 |
| 报告导出 | 导出 HTML/PDF/Markdown 演示报告 | 包含地图截图、搜索方发现、防护方方案、组织方总结 |

### 6.2 搜索方核心功能：目标检测 / 变化检测

| 模块 | 功能 | 演示结果 |
|------|------|----------|
| 目标检测 | 对视频/图片进行地面对象检测 | 检测框、类别、置信度、时间戳、地图点位 |
| 变化检测 | 对比基线与当前影像，识别变化对象或区域状态变化 | 新增/消失/移动/异常区域高亮 |
| 搜索轨迹 | 展示搜索方搜索路线和覆盖进度 | 地图上可见搜索方搜索路径 |
| 发现点复核 | 对候选点确认/驳回/备注 | 生成可解释的发现证据 |
| 搜索方报告 | 导出发现点、变化点、截图、坐标和时间戳 | 证明搜索/识别闭环 |

### 6.3 防护方核心功能：覆盖规划方案设计

| 模块 | 功能 | 演示结果 |
|------|------|----------|
| 覆盖规划 | 绘制保护区、禁入区、观察区、起降点，生成覆盖与采集建议 | 覆盖率、建议观测点、建议路径、预计耗时 |
| 覆盖缺口 | 显示未覆盖区域、盲区、重复覆盖区 | 防护方能解释防护方案质量 |
| 巡查路径 | 生成值守/巡查路径建议 | 地图上的防护方巡查路线 |
| 变化监测 | 在保护区内复用 变化检测，提示变化点 | 防护方视角下的异常区域高亮 |
| 防护方报告 | 导出 覆盖规划 方案、覆盖图、盲区和响应建议 | 证明防护/值守闭环 |

### 6.4 组织方核心功能：定位态势与全局规划

| 模块 | 功能 | 演示结果 |
|------|------|----------|
| 活动配置 | 创建活动区域、搜索/防护队伍、任务边界 | 形成三方演示上下文 |
| 全局定位态势 | 显示搜索方、防护方、无人机轨迹与事件时间线 | 组织方掌握全局态势 |
| 路径规划 | 对搜索方搜索路线和防护方巡查路线进行查看/调整/确认 | 形成“组织方统筹、搜索/防护执行”的闭环 |
| 裁判 / 调度视图 | 查看搜索方发现、防护方覆盖、越界/变化事件 | MVP 可模拟裁判判定 |
| 总报告 | 生成组织方总览报告 | 适合路演展示商业价值 |

### 6.5 明确不做（MVP Exclusions）

- 不做无人机飞控控制。
- 不做自动起飞、自动降落、自动航线下发。
- 不做 LoRa Mesh 实物组网。
- 不做 Jetson/加固箱硬件样机。
- 不做 IP65/MIL-STD-810H/EMC 实测。
- 不做真实 RTK <2cm 精度承诺。
- 不做军事化目标分类、武器识别、打击排序或作战控制。

## 7. 演示脚本

### 7.1 Demo A：Hide and Seek 实景对抗赛（3 分钟融资路演版）

1. 打开组织方视图，创建活动区域、搜索方搜索区、防护方保护/隐藏区和安全边界。
2. 导入无人机视频和飞行日志，组织方 定位态势 视图显示轨迹和时间线。
3. 切换搜索方视图，运行 目标检测/变化检测，地图上出现发现点、变化点和候选列表。
4. 切换防护方视图，基于保护/隐藏区生成 覆盖规划方案、盲区提示和巡查路径。
5. 回到组织方视图，查看搜索方发现与防护方覆盖的叠加态势，生成裁判/调度报告。

### 7.2 Demo B：野外搜救（3 分钟备选版）

1. 组织方创建搜救区域、起点、禁入区和优先搜索区。
2. 搜索方导入无人机视频/图片/日志，运行 目标检测/变化检测，生成疑似人员/动物/风险点。
3. 组织方基于 定位态势 和路径规划重新分配搜索路线。
4. 导出搜救报告和证据包。

### 7.3 8–10 分钟技术演示版

1. 说明组织/搜索/防护三方定义和民用/商用边界。
2. 展示核心场景：Hide and Seek、大型 Milsim、安保训练、野外搜救。
3. 选择硬件方案 A 或 B。
4. 展示组织方：活动配置、定位态势、无人机轨迹、搜索/防护区域。
5. 展示搜索方：目标检测结果如何映射到地图，变化检测如何识别变化。
6. 展示防护方：覆盖规划方案设计、保护区、禁入区、覆盖率、盲区。
7. 展示定位-路径规划：搜索方搜索路径、防护方巡查路径、组织方调度确认。
8. 展示报告导出和审计日志。
9. 说明正式 MSkit 的升级路径：Jetson、RTK、热成像、LoRa、加固箱、认证。

## 9. MVP 验收标准

| 指标 | MVP 验收线 |
|------|------------|
| 演示稳定性 | 连续演示 10 分钟无崩溃 |
| 三方视图 | 组织/搜索/防护三方视图或过滤器可切换 |
| 场景脚本 | Hide and Seek 主脚本可完整演示；野外搜救备选脚本可演示核心流程 |
| 数据接入 | 可导入至少 1 段视频、20 张图片或 1 份飞行日志 |
| 地图显示 | 离线地图加载 <5 秒，轨迹和检测点可显示 |
| 组织方 定位态势 | 可显示活动区、搜索/防护区域、无人机轨迹、事件时间线、路径建议 |
| 搜索方 目标检测/变化检测 | 至少 1–3 类民用目标检测可稳定展示；至少支持两组图像/视频片段变化高亮 |

| 报告导出 | 可导出包含组织方总览、搜索方发现、防护方方案、路径建议的报告 |
| 合规边界 | UI 和脚本不出现军用采购、武器化、作战控制、打击排序等表述 |

## 10. 立项 / 融资展示重点

MVP 路演应强调：

1. **三方架构清晰**：组织方统揽全局，搜索方搜索检测，防护方覆盖防护。
2. **场景落地明确**：Hide and Seek、大型 Milsim、安保训练、野外搜救都复用同一三方底座。
3. **低硬件门槛**：用笔记本和现成无人机即可展示核心价值。
4. **明确升级路径**：从笔记本 MVP 迁移到 Jetson + 加固箱 + RTK + LoRa。
5. **产品护城河**：目标检测/变化检测 + 覆盖规划 + 定位态势/路径规划 + 离线地图 + 报告的三方闭环，而不是单点 AI 检测。
6. **合规边界清晰**：MVP 不做飞控控制，不做武器化或作战用途。
